Transformer / 注意力机制 / 模型架构深度对比

1. Transformer 核心

🔬 费曼类比：翻译团队

想象一个翻译团队正在将英文句子"This is a cat"翻译成中文。传统的做法是逐词翻译——像 RNN 一样，看完第一个词再看第二个，效率低下且容易忘记前面。Transformer 的做法完全不同：每个团队成员（每个词）同时阅读所有词，然后决定自己应该关注谁。

比如"cat"这个词，它在翻译时需要关注"a"（单数冠词）和"This"（主语指示），可能完全不需要关注"is"。每个词为句子中所有词分配注意力权重，然后基于这些加权信息生成自己的表征——这就是 Self-Attention 的精髓。

整个团队分为两组：Encoder（编码团队）负责理解原文，Decoder（解码团队）负责生成译文。编码团队看完整句话，解码团队边看已生成的内容边看原文。

Transformer 架构由 Vaswani 等人在 2017 年提出("Attention Is All You Need")，核心思想是完全放弃循环和卷积，仅依赖注意力机制。

  • Encoder：双向 Self-Attention，每个位置能看见所有位置 → 理解上下文
  • Decoder：Masked Self-Attention（只能看左侧）+ Cross-Attention（看 Encoder 输出）→ 生成文本
  • Feed-Forward：每个位置的独立 MLP，引入非线性变换
  • 残差连接 + LayerNorm：解决深层网络梯度消失，加速训练

2. 注意力机制

2.1 Self-Attention 计算流程

Self-Attention 的核心是每个 token 都通过 Q/K/V 变换与其他所有 token 交互。给定输入序列 \(X\)，计算步骤：

  1. Q/K/V 投影：每个 token 通过三个不同的权重矩阵 \(W_Q, W_K, W_V\) 生成 Query、Key、Value
  2. Score 计算：\(S = Q \cdot K^T / \sqrt{d_k}\)（点积缩放，防止 softmax 进入饱和区）
  3. Softmax 归一化：将 score 转为概率分布，和为 1
  4. 加权求和：用注意力权重对 Value 加权求和得到输出

2.2 Multi-Head Attention

多头注意力将 Q/K/V 分别投影到 h 个不同的子空间，每个头独立计算注意力后再拼接。这就像请 h 个不同的专家同时阅读文本——一个关注语法关系，一个关注语义关联，一个关注指代消解。每个头捕获不同的特征：

  • 头 1：捕捉句法依赖（如主谓关系）
  • 头 2：关注位置邻近性
  • 头 3：捕获语义相似性
  • 头 h：多种混合模式

2.3 Masked Attention

Decoder 中使用的 Masked Self-Attention 通过上三角掩码矩阵，确保第 i 个 token 只能关注 1…i 位置的 token。这是自回归生成的关键——模型在预测下一个词时不能"偷看"未来的词。

3. 位置编码

Self-Attention 本身是置换等变的——打乱输入顺序，输出也随之打乱。没有位置信息，"我打你"和"你打我"在模型眼中一模一样！位置编码就是为了给模型注入序列位置信息。

3.1 Sinusoidal（正弦波编码）

原始 Transformer 使用的固定编码：\(PE_{(pos,2i)} = sin(pos/10000^{2i/d})\)，偶数维用 sin，奇数维用 cos。无需学习，可外推到任意长度，但无法感知相对位置。

3.2 RoPE（旋转位置编码）

RoPE 将位置信息旋转到 Q 和 K 的向量空间中。两个 token 的注意力分数只取决于它们的相对位置差。公式：\(f_q(x_m,m) = (x_m W_q) \cdot e^{im\theta}\)。LLaMA、Mistral、Qwen 均使用 RoPE。

3.3 ALiBi

ALiBi 不做任何位置嵌入，而是在注意力分数上加上一个与距离成正比的负偏置：\(AttentionScore + bias\)，其中 \(bias = -m \cdot |i - j|\)。距离越远的 token，注意力分数被压得越低。计算零开销，可外推至超长序列。

4. 预训练 → 微调 → RLHF 完整训练流水线

现代 LLM 的训练不是一步到位的，而是经历了三个主要阶段，每个阶段都有明确的目标和数据策略。

4.1 预训练（Pre-training）

在 TB 级互联网文本数据上进行自监督学习。目标是预测下一个 token（自回归语言模型），模型学习语法、事实知识、推理模式和世界知识。此阶段的模型称为 Base Model（基础模型），能力全面但不擅长对话——它只是"续写"而非"回答"。

4.2 监督微调（SFT）

使用高质量的指令-回答对进行监督学习。让模型学会理解指令格式、遵循用户意图。数据包括人工标注的问答、思维链推理、代码编写等。SFT 后模型能"好好说话"了。

4.3 RLHF（人类反馈强化学习）

由 OpenAI 在 InstructGPT 论文中提出的经典三步法：

  1. 收集人类偏好数据：让人类对多个模型回答排序
  2. 训练 Reward Model：预测人类偏好的奖励模型
  3. PPO 强化学习：用 RM 的奖励信号进一步优化 SFT 模型

RLHF 的核心是对齐（Alignment）——让模型的行为符合人类价值观，减少有害输出，提高有用性。

5. 主流模型架构对比

5.1 GPT（Decoder-Only） vs Encoder-Decoder vs MoE

📌 Decoder-Only（GPT 系列）

仅由 Decoder 堆叠而成，使用因果注意力（Causal Attention）。每个 token 只能关注左侧。

✅ 生成速度快，架构简单，训练高效
✅ 支持任意长度生成
✅ 适合对话、续写、代码生成
❌ 双向理解能力弱于 Encoder-Decoder

代表：GPT-4, LLaMA, Mistral, Qwen

📌 Encoder-Decoder

完整的 Encoder + Decoder 架构。Encoder 双向编码，Decoder 交叉注意力解码。

✅ 双向理解能力强
✅ 适合翻译、摘要、分类任务
❌ 推理速度慢（两阶段）
❌ 参数量大，训练复杂

代表：T5, BART, GLM, Flan-T5

📌 MoE（混合专家模型）

MoE 在每层 FFN 中放置多个"专家"子网络，通过路由门控让每个 token 只激活部分专家。用更少的计算量获得更大的模型容量。

✅ 同等计算量下更大的模型容量
✅ 推理速度快（稀疏激活）
❌ 显存占用大（需加载全部专家）
❌ 训练稳定性挑战

代表：Mixtral 8×7B, DeepSeek V3, GPT-4, Qwen2.5-MoE

6. MoE 混合专家模型

🔬 费曼类比：专家团队

想象你有一支由 100 位专家组成的团队（每个专家擅长不同领域：数学、写作、编程、翻译……）。每来一个任务，你不需要让所有 100 个人都干活——只需要找到最擅长的 2-3 位专家来完成即可。这就是 MoE 的精髓！

路由器（Router / Gate）是一个轻量级的网络，它看一眼当前 token 的特征，然后说："这个问题交给专家 #5 和 #17"。只有这两个专家的 FFN 被激活，其余 98 个专家处于休眠状态。虽然模型总参数量是巨大的（671B），但每次推理只激活一小部分（约 37B），这就是 MoE 高效的原因。

6.1 DeepSeek V3 / R1 的 MoE 创新

  • Fine-Grained MoE：256 个细粒度专家，每个 token 激活 8 个，专家粒度更细，分工更精
  • Multi-Token Prediction：在训练时同时预测多个未来 token，提高数据效率
  • FP8 混合精度训练：大幅降低训练成本，据称训练成本仅 $5.6M
  • DeepSeek R1：在 V3 基础上通过强化学习（无 SFT）获得超强推理能力，类似 o1 的思维链模式

8. 常见问题与口播总结

❓ Q1：为什么 GPT 只用 Decoder 而不用 Encoder？

GPT 系列使用 Decoder-Only 架构的主要原因是可扩展性和生成效率：

  1. 自回归生成天然适配 Decoder：LLM 的核心任务是逐词生成文本，而 Decoder 的因果注意力正好是从左到右的生成过程
  2. 架构简单，易于扩展：只有一种模块（Decoder），更容易大规模训练和优化
  3. 训练效率高：预训练时所有位置可以并行计算（因为只依赖左侧），而 Encoder-Decoder 需要两阶段处理
  4. 事实证明够用：随着模型规模增大，Decoder-Only 模型在理解任务上并不比 Encoder-Decoder 差——大模型的"涌现能力"弥补了双向理解的缺失

一句话：Decoder-Only 是"大力出奇迹"的最佳实践——足够简单，足够强大。

❓ Q2：Self-Attention 的时间复杂度 O(n²) 太大，如何优化？

确实，Self-Attention 的复杂度是 O(n²)，n 是序列长度。当上下文达到 128K、1M token 时，计算量爆炸。主流优化方法：

  • Flash Attention：通过 IO-aware 的 CUDA kernel 优化，将显存读写从 O(n²) 降到 O(n)，速度提升 2-4 倍。已是所有大模型的标配。
  • 稀疏注意力（Sparse Attention）：只计算部分 token 对之间的注意力，例如局部窗口 + 全局 token。
  • 分组查询注意力（GQA / MQA）：多个 Query 头共享同一组 Key/Value 头，减少 KV Cache 大小。LLaMA 2/3、Mistral 都使用 GQA。
  • Sliding Window Attention：每个 token 只看固定窗口内的邻居（Mistral 使用）。
  • 线性注意力（Linear Attention）：用核方法将复杂度降到 O(n)，但效果略有损失。

目前 Flash Attention + GQA 是实际部署中最实用的组合。

❓ Q3：DeepSeek R1 和 V3 有什么区别？什么时候用 R1？

DeepSeek V3 是基础 MoE 模型（671B 参数），擅长通用对话、写作、翻译等任务。它的训练遵循标准的三阶段流程。

DeepSeek R1 是在 V3 基础上通过强化学习（不含 SFT）专门训练出的推理增强版本。它学会了"思维链"——在回答前先进行内部推理，展示思考过程。

选型建议：

  • ✅ 用 R1：数学证明、逻辑推理、复杂编程、学术分析、需要展示思考过程的任务
  • ✅ 用 V3：日常对话、创意写作、翻译、摘要、简单问答（V3 更快更便宜）
  • 💡 两者可以互补：先用 R1 进行深度分析，再用 V3 润色表达能力
❓ Q4：什么是"上下文窗口"？为什么有些模型支持 128K 甚至 1M？

上下文窗口是模型一次能"看到"的最大 token 数量。就像人的短期记忆——超过这个长度，模型就会"忘记"前面的内容。

限制上下文长度的两个瓶颈：

  1. 计算复杂度：Self-Attention 是 O(n²)，n 翻倍，计算量翻 4 倍
  2. 位置编码外推：如果训练时最长是 4K 的序列，直接输入 128K 的序列，模型可能因为位置编码没见过那么长的距离而迷失

现代模型能支持超长上下文的关键技术：

  • RoPE + 插值：将位置频率进行线性或 NTK 插值，让模型"适应"更长的位置
  • YaRN（Yet another RoPE extensioN）：更精细的 RoPE 扩展方法
  • LongRoPE：DeepSeek 使用的超长上下文扩展技术
  • Flash Attention：让 O(n²) 的计算在 GPU 上变得可行
  • 高质量长文本数据：在训练中加入足够多的长序列样本
❓ Q5：MoE 模型比 Dense 模型更好吗？有什么缺点？

MoE 不是绝对更好，而是一种"算力换容量"的策略。

MoE 的优点：

  • 总参数量可以做得非常大，但每次推理只激活一小部分（稀疏激活）
  • 同等算力下，MoE 模型的效果通常优于相同激活参数量的 Dense 模型
  • 训练更高效，因为每个 token 只更新部分专家

MoE 的缺点：

  • 显存需求大：所有专家参数必须加载到内存中。DeepSeek V3 的 671B 参数需要约 1300GB 显存，必须多卡分布式部署
  • 推理批处理效率：不同 token 可能路由到不同专家，专家负载不均衡影响批处理效率
  • 训练稳定性：门控网络可能崩溃（所有 token 都选同一个专家），需要负载均衡 loss 来约束
  • 推理延迟：专家间的 all-to-all 通信增加了延迟

总结：如果你有足够的 GPU（多卡部署），MoE 是很好的选择。如果你只有单卡（如消费级 24GB），Dense 模型更实用。

❓ Q6：什么是"涌现能力"？为什么小模型没有？

涌现能力（Emergent Abilities）是指模型在规模增大到某个阈值后，突然出现的、在更小的模型中完全不存在的能力。就像水在 100°C 突然沸腾一样——不是逐渐变热，而是质变。

典型涌现能力包括：

  • 上下文学习（In-Context Learning）：给几个例子就能完成新任务
  • 思维链推理（Chain-of-Thought）：能展示解题步骤
  • 指令遵循：理解并执行复杂指令
  • 代码生成：编写可运行的代码

为什么小模型没有？学术界有两种主流解释：

  1. 能力连续性假说：能力其实是连续的，只是小模型的表现太弱（低于评估指标的下限），看起来像"突然出现"
  2. 表征重组假说：当模型达到一定规模后，注意力头之间形成了复杂的协作模式，产生了质变——就像神经元网络在达到临界复杂度后涌现出意识

🎙️ 口播总结

今天我们系统梳理了 LLM 的核心知识体系：

  1. Transformer 是模型的骨架，Encoder 负责理解，Decoder 负责生成，靠注意力机制连接
  2. Self-Attention 通过 Q/K/V 三步计算让每个词和所有词建立关联，多头注意力让模型从多个角度理解文本
  3. 位置编码 解决了注意力机制对顺序不敏感的问题，RoPE 成为当前主流方案
  4. 三阶段训练 从预训练（学知识）→ SFT（学对话）→ RLHF（学对齐），层层递进
  5. 模型架构上 Decoder-Only 是当前主流（GPT/Llama/Qwen），MoE 让大模型更经济（DeepSeek V3/Mixtral）
  6. MoE 通过路由专家机制实现了"大容量、小激活"的高效推理
  7. 选型要看语言、预算、场景——中文首选 DeepSeek/Qwen，复杂推理选 R1/Claude，低成本用小模型

记住一句话：没有最好的模型，只有最适合你的模型。技术选型的核心是理清自己的约束条件（成本、速度、精度、隐私），然后在约束边界内找到最优解。
