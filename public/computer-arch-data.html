🔢 计算机组成原理 · 数据的表示与运算

一句话：计算机内部只有 0 和 1，但用这简单的两个符号，它能表示整数、浮点数、负数、字符——甚至你正在读的这段文字。数据的表示与运算是计算机科学的 "数字化学"——学习 "二进制如何编码有符号整数？IEEE 754 如何表示 0.1？ALU 的内部逻辑是什么？"。本文从进制转换出发，逐步深入到补码、浮点标准、定点运算和 ALU 结构。

🔢 进制转换

🪙 类比：不同语言的"数字"

十进制是我们日常说的"中文"描述数字，二进制是计算机的"机器语"——只有 0 和 1 两个词。八进制（0-7）和十六进制（0-9, A-F）是"方言"，因为 3 位二进制凑 1 位八进制、4 位二进制凑 1 位十六进制，读写起来比二进制串方便得多。

核心口诀：整数部分"除基取余倒序排"，小数部分"乘基取整正序排"。

➖ 原码 / 反码 / 补码

🕹️ 类比：温度计的刻度

想象一个圆形温度计，从 0° 顺时针到 359°，再转一圈回到 0°。补码就像这个圆形温度计——-1 在圆上等价于 359（即 360 - 1）。对于 n 位二进制数，-x 的补码就是 2ⁿ - x。这样可以用同一个加法器处理正数和负数，不需要单独的减法器。

为什么用补码？因为 CPU 不需要设计减法电路，加法和减法统一为加法运算——3 - 5 = 3 + (-5) = 3 + [1011]补 = [1110]补 = -2。硬件更简单、更便宜、更快！

三种编码对比（以一个字节 8 位为例）

十进制原码反码补码说明
+5000001010000010100000101正数：三码相同
-5100001011111101011111011符号位不变，其他取反 → +1
+0000000000000000000000000补码的 +0 唯一表示
-0100000001111111100000000❌ 原码/反码有歧义！补码无歧义
-128❌ 溢出❌ 溢出10000000补码多表示一个最小负数

🔄 补码的"取反加一"推导

公式：对一个 n 位二进制数 x，其补码定义为：

// 补码定义
[x]补 = 
    x                    (0 ≤ x ≤ 2ⁿ⁻¹-1, 正数)
    2ⁿ + x              (-2ⁿ⁻¹ ≤ x < 0, 负数)
    
// 以 n=8, x=-5 为例
[-5]补 = 2⁸ + (-5) = 256 - 5 = 251 = 11111011₂

// 快速计算方法：原码取反加一
  -5 的原码 = 10000101
  取反     = 11111010  （符号位不变，数值位取反）
  +1       = 11111011  ✅

💯 浮点数 IEEE 754 标准

🔬 类比：科学记数法

十进制科学记数法：3.14159 × 10⁰（尾数 × 底数^指数）。IEEE 754 浮点数就是二进制的科学记数法——用 尾数 × 2^指数 表示实数。但因为二进制和有限精度，0.1 + 0.2 ≠ 0.3，这才是"浮点误差"的根源。

核心口诀：符号位 S(1) | 阶码位 E(8或11) | 尾数位 M(23或52)

单精度 (float) vs 双精度 (double)

属性单精度 float (32-bit)双精度 double (64-bit)
总位数3264
符号位 S1 bit (bit 31)1 bit (bit 63)
阶码 E8 bits (bit 30-23)11 bits (bit 62-52)
尾数 M23 bits (bit 22-0)52 bits (bit 51-0)
偏置 (Bias)1271023
真值公式(-1)^S × 2^(E-127) × 1.M(-1)^S × 2^(E-1023) × 1.M
精度~7 位十进制~16 位十进制
范围±1.2×10⁻³⁸ ~ ±3.4×10³⁸±2.2×10⁻³⁰⁸ ~ ±1.8×10³⁰⁸

为什么 0.1 + 0.2 ≠ 0.3？

// JavaScript 中验证
console.log(0.1 + 0.2);   // 0.30000000000000004
console.log(0.1 + 0.2 === 0.3);  // false

// 原因：0.1 和 0.2 在二进制中都是无限循环小数
// 0.1 = 0.000110011001100110011...（无限循环）
// 双精度只有 52 位尾数，截断后产生误差

// 解决方案：使用整数计算或引入误差容限
const EPSILON = 1e-10;
Math.abs((0.1 + 0.2) - 0.3) < EPSILON;  // true

🧮 定点运算

🧮 类比：小时候学的竖式计算

加法的"进位"、减法的"借位"、乘法的"部分积相加"、除法的"试商减去除数"——计算机的定点运算本质上就是 竖式的二进制版。区别在于：计算机用门电路实现，需要优化速度和面积。超前进位加法器用"预测进位"代替"逐位传播"；Booth 算法用编码技巧减少乘法累加次数。

定点加法与减法

类型原理延迟电路复杂度
串行进位加法器 (RCA)每一位的进位等待前一位输出O(n) — 32 位需约 32 个门延迟低
超前进位加法器 (CLA)并行计算所有位的进位O(log n) — 32 位约 4-5 级中高
进位选择加法器预计算进位 0/1 两种结果，用 MUX 选O(√n)高

定点乘法

✖️ Booth 算法原理

传统竖式乘法需要 n 次加法（n 为位数），Booth 算法通过编码 011...110 这类模式减少加法的次数：

// 原始竖式：0101 × 0110（5×6）
      0101  （被乘数）
×     0110  （乘数）
-----------
      0000   ← 0 × 0101
     0101    ← 1 × 0101  ← 左移
    0101     ← 1 × 0101
+   0000     ← 0 × 0101
-----------
=   00011110 = 30 ✅

// Booth 算法：
// 扫描乘数位对 (yᵢ, yᵢ₋₁)，三位决定操作
// 00→无操作 01→+被乘数 10→-被乘数 11→无操作
// 处理乘数 0110（最末补0）：
//   (1,0) → -被乘数 → 1011（补码）
//   (1,1) → 无操作
//   (0,1) → +被乘数 → 0101
// 最终结果一致，但更快处理连续的 1 串

// 每一步：

// 0111 ÷ 0010 (7÷2)

⚙️ ALU 基本结构

🖩 类比：一个高级计算器

ALU (Arithmetic Logic Unit) 就是 CPU 里的 "超级计算器"——给定两个输入 A 和 B，以及操作码 OpCode，它输出计算结果和标志位。加、减、乘、除、与、或、异或、移位……所有你能想到的基本操作都在这里完成。

关键概念：多功能运算器 + 标志寄存器，现代 CPU 还包含桶形移位器、条件码逻辑等增强组件。

图5：ALU 内部结构——输入输出、运算单元、标志位及控制逻辑

1-bit ALU 与 32-bit ALU

🔲 1-bit ALU

最小的 ALU 单元，包含：

  • 全加器（FA）
  • AND / OR 逻辑门
  • MUX 选择器（2选1）
  • 进位输入/输出线
// 1-bit ALU 组合逻辑
Sum   = A ⊕ B ⊕ CarryIn
CarryOut = (A·B) + (A·CarryIn) 
              + (B·CarryIn)
AND   = A · B
OR    = A + B

🔳 32-bit ALU

由 32 个 1-bit ALU 级联：

  • ADD/SUB：通过 SUB 信号控制 B 取反 + CarryIn=1
  • SLT (Set Less Than)：A<B = A - B 的符号位
  • 溢出检测：CarryIn₃₁ ≠ CarryOut₃₁ → OF=1
  • 零检测：所有结果位相或非 → ZF
// RISC-V 中 ADD/SUB 用同一个 ALU
// SUB = ADD 且 B取反 + CarryIn=1
// SLT = SUB 的符号位传给结果最低位

❓ 实战问答

Q1

❓ 为什么计算机用补码而不是原码表示负数？

✅ 三个核心原因：

① 统一加减法电路：补码下 3-5 = 3+(-5) = -2，加法和减法用同一套加法器实现，不需要独立的减法器，硬件更简单。

② 消除 ±0 歧义：原码有 00000000(+0) 和 10000000(-0) 两种 0；补码只有 00000000 一种 0，10000000 表示 -128。

③ 多表示一个数：8 位原码/反码表示范围 -127~+127，补码 -128~+127，多表示一个最小负数。

Q2

❓ 浮点数 0.1 + 0.2 为什么不等于 0.3？如何解决？

✅ 原因：0.1 和 0.2 在二进制中是无限循环小数（0.1 = 0.0001100110011…）IEEE 754 双精度只有 52 位尾数，截断后引入舍入误差。

✅ 解决方案：① 使用整数计算（如货币用分代替元）② 引入误差容限 |a - b| < EPSILON ③ 使用高精度库（如 Python decimal、Java BigDecimal）。

Q3

❓ IEEE 754 中为什么要用偏置（Bias）表示阶码？

✅ 用偏置（单精度 Bias=127）把阶码转换为无符号数，好处是：

① 便于浮点比较：阶码大的数一定大，直接用无符号整数比较器即可，不需要处理负数阶码。

② 简化硬件：浮点加法时阶码对齐、规格化等操作直接用整数加减法处理。

Q4

❓ 超前进位加法器（CLA）为什么比串行进位加法器（RCA）快？

✅ RCA 的进位像水波一样逐位传播，32 位加法需要 32 个门延迟。CLA 通过 Gᵢ = Aᵢ·Bᵢ（生成进位）和 Pᵢ = Aᵢ⊕Bᵢ（传播进位）两个信号，并行计算所有位的进位，只需要 4~5 级门延迟，速度提升约 8 倍。代价是电路复杂度增加。

Q5

❓ 如何判断定点加法是否溢出？

✅ 两种判断方法：

① 双符号位法：使用两位符号位，运算后两位符号位不同则溢出（01 为正溢出，10 为负溢出）。

② 进位异或法：最高位进位和次高位进位不同则溢出。即 Overflow = CarryIn₃₁ ⊕ CarryOut₃₁。

Q6

❓ 什么是 NaN？浮点运算中 NaN 的传递规则是什么？

✅ NaN (Not a Number) 是 IEEE 754 中阶码全为 1、尾数不为 0 的特殊值（如 0/0、∞-∞、sqrt(-1)）。

传递规则：任何包含 NaN 的运算结果都是 NaN。即 NaN + x = NaN、NaN * x = NaN。比较运算中 NaN 与任何值（包括自身）比较都返回 false——这就是为什么 NaN == NaN 为 false。

「数据的表示与运算」核心要点（2 分钟版）

🧩 第一：进制转换。计算机只认二进制，但为了方便阅读和书写，我们常用八进制（3位一组）和十六进制（4位一组）。核心方法：整数除基取余倒序排，小数乘基取整正序排。

🧩 第二：原码反码补码。这是计算机表示有符号数的方案，补码是终极答案。它通过"取反加一"把负数编码，让加法和减法共用同一个电路，而且没有 ±0 的问题。记住：-x 的补码就是 2ⁿ - x。

🧩 第三：IEEE 754 浮点标准。浮点数 = 符号位 + 阶码（带偏置）+ 尾数（隐含 1.）。单精度偏移 127，双精度偏移 1023。0.1 + 0.2 ≠ 0.3 的本质是无理数在有限精度下的舍入。

🧩 第四：定点运算。加法器从串行进位（慢但简单）进化到超前进位（快但复杂）。乘法用 Booth 算法优化，除法用恢复/不恢复余数法。

🧩 第五：ALU 结构。ALU = 加法器 + 逻辑单元 + 移位器 + 多路选择器 + 标志位输出。OpCode 决定做什么操作，标志位为条件分支提供依据。
