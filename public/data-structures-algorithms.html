1. 复杂度分析

1.1 时间复杂度 — 大 O 表示法

大 O 表示法描述算法运行时间随输入规模增长的增长趋势。常见复杂度从快到慢：

O(1) O(logN) O(N) O(NlogN) O(N²)

1.2 最好 / 最坏 / 平均 / 均摊

类型含义示例（快排）
最好输入最理想时的复杂度O(NlogN) — 每次 pivot 正好中位数
最坏输入最差时的复杂度O(N²) — 已有序数组 + 首元素 pivot
平均所有输入概率加权O(NlogN)
均摊连续操作的平摊代价动态数组扩容：每次 append 均摊 O(1)

1.3 空间复杂度

衡量算法运行时额外使用的内存空间。

  • 原地算法 (In-place)：空间复杂度 O(1) — 如冒泡排序
  • 额外空间：如归并排序需要 O(N) 辅助数组
  • 递归栈空间：如快排递归深度平均 O(logN)，最坏 O(N)

📝 费曼要点 — 复杂度

2.2 队列 FIFO & 堆栈 LIFO

📌 Go 中的栈和队列

Go 没有内置的栈/队列容器，通常用 slice 模拟：

// 栈 (LIFO)
stack := make([]int, 0)
stack = append(stack, 1)     // push
top := stack[len(stack)-1]    // peek
stack = stack[:len(stack)-1] // pop

// 队列 (FIFO)  — 用 slice 模拟，或用 list 库
queue := make([]int, 0)
queue = append(queue, 1)     // enqueue
front := queue[0]             // peek
queue = queue[1:]              // dequeue (O(1) for slice ref)
⚠️ 大量 dequeue 时 slice 头部 pop 会导致内存泄漏，可用 ring buffer 或 container/list

2.3 经典题：链表反转

迭代法 — 三指针（prev, curr, next）

type ListNode struct {
    Val  int
    Next *ListNode
}

// 迭代反转
func reverseListIter(head *ListNode) *ListNode {
    var prev *ListNode
    curr := head
    for curr != nil {
        next := curr.Next  // 保存下一个
        curr.Next = prev   // 反转指针
        prev = curr        // 前进
        curr = next
    }
    return prev  // 新头
}

递归法 — 更优雅但使用 O(N) 栈空间

func reverseListRecur(head *ListNode) *ListNode {
    if head == nil || head.Next == nil {
        return head
    }
    newHead := reverseListRecur(head.Next)
    head.Next.Next = head  // 反转当前节点
    head.Next = nil
    return newHead
}

2.4 更多链表高频题

🔗 链表必刷题
// 环检测（快慢指针）
func hasCycle(head *ListNode) bool {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        if slow == fast { return true }
    }
    return false
}

// 合并有序链表
func mergeTwoLists(l1, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    cur := dummy
    for l1 != nil && l2 != nil {
        if l1.Val < l2.Val { cur.Next = l1; l1 = l1.Next
        } else { cur.Next = l2; l2 = l2.Next }
        cur = cur.Next
    }
    if l1 != nil { cur.Next = l1 }
    if l2 != nil { cur.Next = l2 }
    return dummy.Next
}

// 找中间节点（快慢指针）
func middleNode(head *ListNode) *ListNode {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }
    return slow
}

3. 字符串匹配

3.1 KMP 算法

KMP 利用 next 数组（部分匹配表）避免暴力回溯，将匹配时间降到 O(N+M)。

// 构建 next 数组
func buildNext(pat string) []int {
    n := len(pat)
    next := make([]int, n)
    j := 0
    for i := 1; i < n; i++ {
        for j > 0 && pat[i] != pat[j] { j = next[j-1] }
        if pat[i] == pat[j] { j++; next[i] = j }
    }
    return next
}

// KMP 匹配
func kmpSearch(txt, pat string) []int {
    if len(pat) == 0 { return []int{0} }
    next := buildNext(pat)
    res := []int{}
    j := 0
    for i := 0; i < len(txt); i++ {
        for j > 0 && txt[i] != pat[j] { j = next[j-1] }
        if txt[i] == pat[j] { j++ }
        if j == len(pat) {
            res = append(res, i-j+1)
            j = next[j-1]
        }
    }
    return res
}

3.2 Trie 字典树

Trie（前缀树）用于高效存储和检索字符串集合，适合自动补全、拼写检查。

type TrieNode struct {
    children [26]*TrieNode
    isEnd    bool
}

type Trie struct { root *TrieNode }

func Constructor() Trie {
    return Trie{root: &TrieNode{}}
}

func (t *Trie) Insert(word string) {
    node := t.root
    for _, ch := range word {
        idx := ch - 'a'
        if node.children[idx] == nil {
            node.children[idx] = &TrieNode{}
        }
        node = node.children[idx]
    }
    node.isEnd = true
}

func (t *Trie) Search(word string) bool {
    node := t.root
    for _, ch := range word {
        idx := ch - 'a'
        if node.children[idx] == nil { return false }
        node = node.children[idx]
    }
    return node.isEnd
}

func (t *Trie) StartsWith(prefix string) bool {
    node := t.root
    for _, ch := range prefix {
        idx := ch - 'a'
        if node.children[idx] == nil { return false }
        node = node.children[idx]
    }
    return true
}

3.3 AC 自动机

AC 自动机 = Trie + fail 指针，用于多模式串匹配。在 Trie 树上构建失败指针（类似 KMP 的 next），匹配时沿 fail 指针回溯。

  • 构建：BFS 遍历 Trie 树，为每个节点设置 fail 指针
  • 匹配：遍历文本，用 fail 指针处理失配
  • 复杂度：O(N + M + K) — N 文本长，M 模式串总长，K 匹配次数
📝 费曼要点 — 字符串匹配

暴力匹配 O(NM) → KMP O(N+M) 用 next 数组避免回溯 → Trie O(总长) 前缀匹配 → AC 自动机多模式匹配。KMP 掌握 next 数组构建，Trie 掌握 26 叉树 + isEnd 标记。

4. 排序与二分

4.1 二分查找模板

左闭右闭 [left, right] — 最常用模板

func binarySearch(nums []int, target int) int {
    left, right := 0, len(nums)-1
    for left <= right {
        mid := left + (right-left)/2  // 防溢出
        if nums[mid] == target { return mid }
        if nums[mid] < target { left = mid + 1
        } else { right = mid - 1 }
    }
    return -1
}

左闭右开 [left, right) — 适用于 lower_bound

func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums)
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] < target { left = mid + 1
        } else { right = mid }
    }
    return left  // 第一个 ≥ target 的位置
}

4.2 快速排序

核心思路：选 pivot → 分区（左边 ≤ pivot，右边 ≥ pivot）→ 递归左右。

func quickSort(arr []int, left, right int) {
    if left >= right { return }
    pivot := partition(arr, left, right)
    quickSort(arr, left, pivot-1)
    quickSort(arr, pivot+1, right)
}

func partition(arr []int, left, right int) int {
    pivot := arr[right]  // 选最后一个元素为 pivot
    i := left
    for j := left; j < right; j++ {
        if arr[j] < pivot {
            arr[i], arr[j] = arr[j], arr[i]
            i++
        }
    }
    arr[i], arr[right] = arr[right], arr[i]
    return i
}

5. 散列表 / Hash

5.1 哈希算法 & 冲突解决

哈希表通过哈希函数将 key 映射到数组索引，实现近似 O(1) 的查找。

冲突解决方式原理优点缺点
拉链法 (Chaining)每个槽挂一个链表/红黑树实现简单，负载因子可高额外指针开销
开放地址法冲突时寻找下一个空位无指针，缓存友好删除麻烦，性能退化快

5.2 动态扩容 & 渐进式 Rehash

当负载因子（元素数/桶数）超过阈值时，扩容 2 倍并 rehash。

  • 一次性 rehash：全部重算，会有瞬间卡顿
  • 渐进式 rehash：将 rehash 分摊到每次操作，每次搬移少量桶
  • Redis dict 使用渐进式 rehash，维护两张哈希表

5.4 手写 Hash 表

type KVPair struct {
    key   string
    value int
}

type HashMap struct {
    buckets [][]KVPair
    size    int
}

const loadFactor = 0.75

func NewHashMap(cap int) *HashMap {
    return &HashMap{buckets: make([][]KVPair, cap)}
}

func (m *HashMap) hash(key string) int {
    h := 0
    for _, ch := range key {
        h = h*31 + int(ch)
    }
    return h % len(m.buckets)
}

func (m *HashMap) Put(key string, val int) {
    idx := m.hash(key)
    for i, kv := range m.buckets[idx] {
        if kv.key == key { m.buckets[idx][i].value = val; return }
    }
    m.buckets[idx] = append(m.buckets[idx], KVPair{key, val})
    m.size++
    if float64(m.size)/float64(len(m.buckets)) > loadFactor {
        m.resize()
    }
}

func (m *HashMap) Get(key string) (int, bool) {
    idx := m.hash(key)
    for _, kv := range m.buckets[idx] {
        if kv.key == key { return kv.value, true }
    }
    return 0, false
}

func (m *HashMap) resize() {
    old := m.buckets
    m.buckets = make([][]KVPair, len(old)*2)
    m.size = 0
    for _, bucket := range old {
        for _, kv := range bucket {
            m.Put(kv.key, kv.value)
        }
    }
}

6. 跳表 Skip List

6.1 跳表原理

跳表是在有序链表上增加多层索引，实现 O(logN) 的查找/插入/删除。每层索引以概率 p（通常 1/2）向上提升。

  • 查找：从最高层开始，向右 + 向下
  • 插入：找到位置后插入底层，随机决定是否提升到上层
  • 删除：各层删除对应的节点

6.2 Redis 为什么用跳表

  • 范围查询：跳表在有序链表上可以轻松范围遍历（ZRANGE），红黑树需要中序遍历
  • ZRANK：跳表可以记录跨度（span），O(logN) 计算排名
  • 实现简单：跳表代码量远小于红黑树
  • 并发友好：跳表可以细粒度加锁

6.3 跳表 vs 红黑树

特性跳表红黑树
实现难度易（~200行）难（~1000行）
范围查询O(logN+K) 天然支持O(logN+K) 需中序遍历
平均查询O(logN)O(logN)
内存占用略高（多层指针）较低（2个指针+颜色）
并发性能易实现无锁难实现无锁

6.4 Go 跳表实现（核心）

import (
    "math/rand"
    "time"
)

const maxLevel = 16
const p = 0.5

type skipNode struct {
    val  int
    next []*skipNode
}

type SkipList struct {
    head  *skipNode
    level int
    rng   *rand.Rand
}

func NewSkipList() *SkipList {
    return &SkipList{
        head:  &skipNode{next: make([]*skipNode, maxLevel)},
        level: 1,
        rng:   rand.New(rand.NewSource(time.Now().UnixNano())),
    }
}

func (sl *SkipList) randomLevel() int {
    lvl := 1
    for sl.rng.Float64() < p && lvl < maxLevel { lvl++ }
    return lvl
}

func (sl *SkipList) Insert(val int) {
    update := make([]*skipNode, maxLevel)
    cur := sl.head
    for i := sl.level - 1; i >= 0; i-- {
        for cur.next[i] != nil && cur.next[i].val < val {
            cur = cur.next[i]
        }
        update[i] = cur
    }
    lvl := sl.randomLevel()
    if lvl > sl.level {
        for i := sl.level; i < lvl; i++ { update[i] = sl.head }
        sl.level = lvl
    }
    newNode := &skipNode{val: val, next: make([]*skipNode, lvl)}
    for i := 0; i < lvl; i++ {
        newNode.next[i] = update[i].next[i]
        update[i].next[i] = newNode
    }
}

func (sl *SkipList) Search(val int) bool {
    cur := sl.head
    for i := sl.level - 1; i >= 0; i-- {
        for cur.next[i] != nil && cur.next[i].val < val {
            cur = cur.next[i]
        }
    }
    cur = cur.next[0]
    return cur != nil && cur.val == val
}

7. 二叉树

7.1 二叉树基础

平衡二叉树：左右子树高度差不超过 1（如 AVL、红黑树）
完全二叉树：除最后一层外全满，最后一层左对齐（如堆）

7.2 AVL 树 — 四种旋转

情况条件旋转方式
LL左子树的左子树插入右旋
RR右子树的右子树插入左旋
LR左子树的右子树插入左旋 → 右旋
RL右子树的左子树插入右旋 → 左旋

7.3 红黑树 — 5 条性质

  1. 每个节点非红即黑
  2. 根节点是黑色
  3. 叶子（NIL）是黑色
  4. 红色节点的子节点必为黑色（不能有连续红）
  5. 任意节点到叶子的所有路径，黑色节点数量相同
💡 为什么 Java TreeMap 用红黑树？

红黑树通过「近似平衡」保证 O(logN) 最坏情况，插入/删除只需 O(1) 旋转（AVL 需 O(logN)）。插入频繁 > 查询频繁的场景红黑树更优。

8. 多路查找树 B/B+ 树

8.1 B 树 vs B+ 树

特性B树B+树
内部节点存数据✅ 是❌ 仅存索引（键）
叶子节点存数据✅ 部分叶子✅ 所有数据

8.2 MySQL 为什么用 B+ 树

  • 磁盘 IO 友好：每个节点存储更多键（16KB 页），树高度低（3~4 层即可存千万级数据）
  • 范围查询高效：叶子链表连接，不用多次回溯
  • 查询稳定：总是到叶子层，IO 次数固定
  • 聚簇索引：InnoDB 的聚簇索引叶子存整行数据，非聚簇索引叶子存主键

9. 堆 Heap

9.1 大顶堆 & 小顶堆

堆是一棵完全二叉树，满足父节点 ≥ 子节点（大顶堆）或 ≤ 子节点（小顶堆）。

  • 建堆 heapify：从最后一个非叶子节点开始下沉，O(N)
  • 上浮 (siftUp)：插入时，新节点和父节点比较并交换
  • 下沉 (siftDown)：删除堆顶后，将最后一个元素移到堆顶并下沉

9.2 Go container/heap

import "container/heap"

// 小顶堆实现
type MinHeap []int
func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h MinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *MinHeap) Push(x any)    { *h = append(*h, x.(int)) }
func (h *MinHeap) Pop() any {
    old := *h; n := len(old); x := old[n-1]; *h = old[:n-1]; return x
}

// 使用
h := &MinHeap{3, 1, 4, 1, 5}
heap.Init(h)
heap.Push(h, 2)
for h.Len() > 0 {
    fmt.Print(heap.Pop(h))  // 1 1 2 3 4 5
}

9.3 应用：TopK & 中位数

// TopK (前 K 大) — 小顶堆
func topK(nums []int, k int) []int {
    h := &MinHeap{}
    for _, v := range nums {
        heap.Push(h, v)
        if h.Len() > k { heap.Pop(h) }
    }
    res := make([]int, h.Len())
    for i := len(res)-1; i >= 0; i-- {
        res[i] = heap.Pop(h).(int)
    }
    return res
}

    } else {
        heap.Push(mf.large, num)
    }
    // 平衡两个堆的大小
    if mf.small.Len() > mf.large.Len() + 1 {
        heap.Push(mf.large, heap.Pop(mf.small))
    } else if mf.large.Len() > mf.small.Len() {
        heap.Push(mf.small, heap.Pop(mf.large))
    }
}

func (mf *MedianFinder) FindMedian() float64 {
    if mf.small.Len() > mf.large.Len() {
        return float64((*mf.small)[0])
    }
    return float64((*mf.small)[0] + (*mf.large)[0]) / 2.0
}

10.1 核心思想

🧩 DP 三步法
  1. 定义状态：dp[i] 表示什么？
  2. 状态转移方程：dp[i] 如何从之前的 dp 算出？
  3. 初始化 + 遍历顺序

10.2 0-1 背包问题

// 0-1背包：物品只能选或不选，求容量W下最大价值
// 状态: dp[j] = 容量j能装的最大价值
func zeroOneKnapsack(weights, values []int, W int) int {
    dp := make([]int, W+1)
    for i := 0; i < len(weights); i++ {
        // 逆序遍历避免重复选同一个物品
        for j := W; j >= weights[i]; j-- {
            val := dp[j-weights[i]] + values[i]
            if val > dp[j] { dp[j] = val }
        }
    }
    return dp[W]
}

// 完全背包：物品无限数量，正序遍历
func completeKnapsack(weights, values []int, W int) int {
    dp := make([]int, W+1)
    for i := 0; i < len(weights); i++ {
        for j := weights[i]; j <= W; j++ {  // 正序！
            val := dp[j-weights[i]] + values[i]
            if val > dp[j] { dp[j] = val }
        }
    }
    return dp[W]
}

10.3 DP vs 贪心 vs 分治

📝 费曼要点 — DP 辨识

遇到「求最值」且「子问题重叠」→ 用 DP。贪心是 DP 的特例（贪心选择性质）。分治是子问题独立不重叠（如归并排序），DP 是子问题重叠（如斐波那契）。

11.1 回溯 — 八皇后

func solveNQueens(n int) [][]string {
    res := [][]string{}
    board := make([][]byte, n)
    for i := 0; i < n; i++ {
        row := make([]byte, n)
        for j := 0; j < n; j++ { row[j] = '.' }
        board[i] = row
    }
    cols := make([]bool, n)
    diag1 := make([]bool, 2*n)  // r+c
    diag2 := make([]bool, 2*n)  // r-c+n
    var backtrack func(r int)
    backtrack = func(r int) {
        if r == n {
            sol := make([]string, n)
            for i := 0; i < n; i++ { sol[i] = string(board[i]) }
            res = append(res, sol)
            return
        }
        for c := 0; c < n; c++ {
            if cols[c] || diag1[r+c] || diag2[r-c+n] { continue }
            board[r][c] = 'Q'
            cols[c], diag1[r+c], diag2[r-c+n] = true, true, true
            backtrack(r + 1)
            board[r][c] = '.'
            cols[c], diag1[r+c], diag2[r-c+n] = false, false, false
        }
    }
    backtrack(0)
    return res
}

11.2 DFS vs BFS 模板

// DFS（递归/栈）
func dfs(graph [][]int, start int, visited []bool) {
    visited[start] = true
    fmt.Printf("访问 %d\n", start)
    for _, neighbor := range graph[start] {
        if !visited[neighbor] {
            dfs(graph, neighbor, visited)
        }
    }
}

// BFS（队列）
func bfs(graph [][]int, start int) {
    visited := make([]bool, len(graph))
    queue := []int{start}
    visited[start] = true
    for len(queue) > 0 {
        cur := queue[0]
        queue = queue[1:]
        fmt.Printf("访问 %d\n", cur)
        for _, neighbor := range graph[cur] {
            if !visited[neighbor] {
                visited[neighbor] = true
                queue = append(queue, neighbor)
            }
        }
    }
}
特性DFSBFS
数据结构栈（递归/显式）队列
空间O(树高)O(宽度)
最短路径❌ 不保证✅ 保证
适用拓扑排序、连通性最短路径、层序遍历

11.3 A* 启发式搜索

A* 使用 f(n) = g(n) + h(n)，其中 g(n) 是起点到 n 的实际代价，h(n) 是 n 到终点的估算代价（启发函数）。

  • h(n) ≤ 真实距离 → A* 保证最优（可采纳）
  • 常见启发函数：曼哈顿距离、欧几里得距离
  • 比 Dijkstra 更快（Dijkstra 是 h(n)=0 的特例）

var memo = make(map[int]int)

func fib(n int) int {
    if n <= 1 { return n }

    memo[n] = fib(n-1) + fib(n-2)

}

12.2 拓扑排序

// Kahn 算法（BFS）
func topoSort(n int, edges [][]int) []int {
    graph := make([][]int, n)
    inDeg := make([]int, n)
    for _, e := range edges {
        graph[e[0]] = append(graph[e[0]], e[1])
        inDeg[e[1]]++
    }
    queue := []int{}
    for i := 0; i < n; i++ {
        if inDeg[i] == 0 { queue = append(queue, i) }
    }
    res := []int{}
    for len(queue) > 0 {
        u := queue[0]; queue = queue[1:]
        res = append(res, u)
        for _, v := range graph[u] {
            inDeg[v]--
            if inDeg[v] == 0 { queue = append(queue, v) }
        }
    }
    if len(res) != n { return []int{} }  // 有环
    return res
}

12.3 Dijkstra 最短路径

import ("container/heap"; "math")

type Item struct {
    node, dist int
}
type PriorityQueue []Item
func (pq PriorityQueue) Len() int { return len(pq) }
func (pq PriorityQueue) Less(i, j int) bool { return pq[i].dist < pq[j].dist }
func (pq PriorityQueue) Swap(i, j int) { pq[i], pq[j] = pq[j], pq[i] }
func (pq *PriorityQueue) Push(x any) { *pq = append(*pq, x.(Item)) }
func (pq *PriorityQueue) Pop() any {
    old := *pq; n := len(old); it := old[n-1]; *pq = old[:n-1]; return it
}

func dijkstra(graph [][][]int, start int) []int {
    n := len(graph)
    dist := make([]int, n)
    for i := 0; i < n; i++ { dist[i] = math.MaxInt32 }
    dist[start] = 0
    pq := &PriorityQueue{}
    heap.Init(pq)
    heap.Push(pq, Item{start, 0})
    for pq.Len() > 0 {
        cur := heap.Pop(pq).(Item)
        if cur.dist > dist[cur.node] { continue }
        for _, edge := range graph[cur.node] {
            to, w := edge[0], edge[1]
            nd := cur.dist + w
            if nd < dist[to] {
                dist[to] = nd
                heap.Push(pq, Item{to, nd})
            }
        }
    }
    return dist
}
📌 Floyd 算法

多源最短路径，O(V³)。核心：dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])，压缩后：

for k := 0; k < n; k++ {
    for i := 0; i < n; i++ {
        for j := 0; j < n; j++ {
            if dist[i][k] + dist[k][j] < dist[i][j] {
                dist[i][j] = dist[i][k] + dist[k][j]
            }
        }
    }
}

13. 其他经典结构

13.1 位图 Bitmap & 布隆过滤器

位图：用 bit 数组表示集合，节省内存。1 个 byte 能标记 8 个元素是否存在。

布隆过滤器：使用 k 个哈希函数 + 位数组，判断元素一定不在或可能在集合中。

  • 优点：空间极省，O(k) 查询
  • 缺点：有误判率（假阳性），不能删除
  • 应用：Redis 缓存穿透防护、垃圾邮件过滤

13.2 并查集 Union-Find

type UnionFind struct {
    parent []int
    rank   []int
}

func NewUnionFind(n int) *UnionFind {
    parent := make([]int, n)
    rank := make([]int, n)
    for i := 0; i < n; i++ { parent[i] = i; rank[i] = 1 }
    return &UnionFind{parent, rank}
}

// 路径压缩
func (uf *UnionFind) Find(x int) int {
    if uf.parent[x] != x {
        uf.parent[x] = uf.Find(uf.parent[x])  // 路径压缩
    }
    return uf.parent[x]
}

// 按秩合并
func (uf *UnionFind) Union(x, y int) {
    rx, ry := uf.Find(x), uf.Find(y)
    if rx == ry { return }
    if uf.rank[rx] < uf.rank[ry] {
        uf.parent[rx] = ry
    } else if uf.rank[rx] > uf.rank[ry] {
        uf.parent[ry] = rx
    } else {
        uf.parent[ry] = rx
        uf.rank[rx]++
    }
}

func (uf *UnionFind) Connected(x, y int) bool {
    return uf.Find(x) == uf.Find(y)
}

// 应用：朋友圈数量、岛屿数量、Kruskal 最小生成树
🔧 并查集复杂度

路径压缩 + 按秩合并后，单次操作近似 O(1)（反阿克曼函数 α(N)）。理论最坏 O(α(N))，实际几乎常数。

14. QA + 总结

❓ Q1: 编程时如何选择数据结构？

费曼回答：先确定你的核心操作是什么——查找、插入、删除、范围查询？

  • 快速查找 → 哈希表（无序）或二叉查找树（有序）
  • 范围查询 → B+树 或 跳表
  • FIFO 顺序处理 → 队列
  • 优先级处理 → 堆
  • 需要回溯 → 栈
  • 图关系 → 邻接表
  • 判断连通性 → 并查集
❓ Q2: 面试中算法题的思考框架？

三步走：

  1. 暴力想出来 → 先确保正确性
  2. 分析瓶颈 → 哪一步最慢？（嵌套循环？重复计算？）
  3. 针对性优化 → 空间换时间（哈希）、分治、DP、双指针

例：两数之和。暴力 O(N²) → 用哈希表 O(N)。

❓ Q3: 什么时候不能用哈希表？
  • 需要有序遍历 — 用红黑树/TreeMap
  • 需要范围查询 — 用 B+树/跳表
  • 需要前缀匹配 — 用 Trie
  • 内存极度受限且可接受误判 — 用布隆过滤器
  • 需要最小/最大值 — 用堆
❓ Q4: 回溯 vs DP vs 贪心怎么区分？
回溯DP贪心
核心枚举所有可能最优子结构 + 重叠子问题局部最优
确定性全部解全局最优可能不是最优
复杂度指数级多项式线性/对数
剪枝可以剪枝优化状态表天然剪枝不需要剪枝
❓ Q5: Go 中 map 遍历为什么是无序的？

Go map 底层是哈希表，扩容后桶的位置会重排。为了强制开发者不依赖遍历顺序，Go 在遍历时引入了随机起始位置。如果需要有序，用 sort.Strings(keys) 或使用第三方有序 map。
