🔌 总线与I/O系统

一句话：总线是计算机的"高速公路"——CPU、内存、I/O设备通过总线交换数据和命令；I/O系统则是"收费站与物流系统"，管理设备和CPU之间的数据流动。本文从总线分类 → 仲裁 → 中断 → DMA → I/O接口，系统梳理计算机内部的数据通路与设备协作机制。

章节核心问题一句话回答
① 总线结构总线分哪些种类？各自干什么？数据/地址/控制三总线 + 系统/IO/通信三级总线体系
② 总线仲裁多个设备要同时用总线怎么办？链式查询（菊花链）/计数器查询/独立请求三种仲裁方式
③ 中断系统设备怎么"通知"CPU？中断请求→判优→响应→保护断点→中断服务→恢复
④ DMA大数据传输怎么不占用CPU？DMA控制器直接掌管总线，批量传输数据，CPU只需启动和收尾
⑤ I/O接口CPU怎么访问设备？统一编址（内存映射）vs 独立编址（专用I/O指令）

🔌 总线分类与特征——计算机的"高速公路网"

🏙️ 类比：城市交通系统

总线就像城市交通网络：数据总线 = 双向车道（数据可以在两个方向走）；地址总线 = 路牌/门牌号（单向，CPU告诉内存要去哪个地址）；控制总线 = 红绿灯+交警手势（读写/中断/复位等控制信号）。

而 系统总线 是城市主干道（连接CPU+内存+IO），IO总线 是专用高速道（PCIe），通信总线 是城际高速公路（USB/以太网）。

图1：总线分级结构 —— 系统总线连接CPU/内存/IO接口，I/O总线连接各类外设，通信总线实现外部互联。底部显示三总线信号特性。

三总线详细对比

总线类型方向宽度 (常见)功能
数据总线 (DB)双向32/64 bits传输数据信息，宽度=CPU一次能处理的数据位数
地址总线 (AB)单向 (CPU→设备)32/48 bits传输地址信息，宽度决定最大寻址空间 (2^n)
控制总线 (CB)双向8~16 bits传输控制/状态信号（读/写/中断/复位/时钟）

总线分级体系

总线等级典型标准连接对象传输速率
系统总线FSB / HT / Infinity FabricCPU ↔ 内存 ↔ I/O桥~50~200 GB/s
I/O总线PCIe 5.0 / SATA / USB 4主控器 ↔ 外设~1~64 GB/s
通信总线Ethernet / RS-232 / SPI计算机 ↔ 外部设备/网络~100 Kbps~100 Gbps

🔍 总线宽度与带宽

总线带宽 = 总线宽度 × 时钟频率 × 传输次数/周期。

例如：64位数据总线，800MHz时钟，DDR（双沿传输）：
带宽 = 64bit/8 × 800MHz × 2 = 8 × 800 × 2 = 12.8 GB/s

现代CPU内部普遍采用片内总线（如Intel的Ring Bus、Mesh架构）连接多个核心和L3缓存，速度远超系统总线限制。

⚖️ 总线仲裁——多个设备争用总线怎么办？

🚦 类比：十字路口的三种交通管理方式

总线仲裁就是管理多个设备"谁先用总线"的问题，类似十字路口管理：
① 链式查询 = 一辆一辆依次放行的\"单向轮转\"（简单但公平性差，靠前设备总优先）；
② 计数器查询 = 电子显示屏轮番显示\"下一辆可以走\"（可以调整优先级，更灵活）；
③ 独立请求 = 每个方向都有独立红绿灯和最优先级的\"特权车\"信号（最快最复杂，适合高性能系统）。

图2：三种总线仲裁方式对比 —— 链式查询（简单可靠，优先级固定）、计数器查询（灵活可编程）、独立请求（高性能，连线复杂）

三种仲裁方式对比总结

特性链式查询计数器查询独立请求
优先级固定，设备号越小越优先可编程（计数器初值）最灵活（由仲裁器决定）
连线数量最少（3条公用线）中等（需log₂N条地址线）最多（2N条专用线）
容错性差（一设备故障，链断）好（可跳过故障设备）好（独立通道）
速度慢（链式传递延迟）中等（计数查询轮次）最快（并行请求）
适用场景简单嵌入式系统中型系统高性能多处理器系统

🚨 中断系统——设备如何\"通知\"CPU？

🔔 类比：餐厅里的\"呼叫服务员\"系统

想象你在餐厅吃饭需要服务：
轮询（Polling） = 服务员每隔30秒来问一遍\"需要什么吗？\"——浪费服务员时间（CPU资源）；
中断（Interrupt） = 你按桌上的呼叫按钮，服务员\按钮)过来——不按就不打扰，高效！
中断优先级 = 同时多个客人按铃，先处理VIP客户（高优先级），低优先级等一等。
中断屏蔽 = 服务员在做重要事情（如结账），暂时忽略呼叫铃，干完再响应。

图3：完整中断响应流程 —— 请求→判优→响应→保护断点→中断服务→恢复断点。底部展示中断优先级层级与屏蔽机制

中断处理的关键概念

概念说明硬件/软件实现
中断请求 (INTR)设备向CPU发送中断信号硬件 — 中断请求线
中断判优多个中断按优先级排队硬件 — 优先级编码器 / 菊花链
中断响应CPU发INTA信号，关中断硬件 — 自动完成
保护断点保存PC、PSW等寄存器到栈硬件自动 + 软件保存其余
中断向量指向中断服务程序的入口地址表硬件 — 中断向量表 (IVT)
中断屏蔽通过屏蔽字禁止某些中断软硬件结合 — PSW中的IF位 / 中断屏蔽寄存器
中断嵌套高优先级中断打断低优先级服务软件 — 开中断后允许嵌套

🔍 中断优先级动态调整：中断屏蔽字

每个中断级都有一个中断屏蔽字（Interrupt Mask），指定该中断服务期间允许哪些更高级的中断来打断。

例如一个4级中断系统：
- 级别1（最高）正在服务 → 屏蔽全部（全1），不让任何中断打断
- 级别3正在服务 → 屏蔽级别1、2、3，但让级别4（更低优先级？不，实际常见的是高优先级能打断低优先级）

正确的嵌套规则：新中断的优先级 > 当前服务程序的优先级 时，才发生中断嵌套。屏蔽字一般设计为屏蔽本级及更低级，允许更高级中断进入。

🔄 轮询 (Polling)

  • CPU定期检查每个设备状态
  • 浪费CPU时间（即使没事件也要查）
  • 响应时间 = 轮询周期（不可控）
  • 实现简单，适合设备少的系统

⚡ 中断驱动 (Interrupt-Driven)

  • 设备有事件才通知CPU
  • CPU利用率高（无事做可执行其他任务）
  • 响应时间快（微秒级）
  • 需中断控制器和复杂处理逻辑

💾 DMA——直接存储器访问，让CPU\"偷懒\"

📦 类比：餐厅里的\"自动补给系统\"

想象餐厅的库存管理：
无DMA（程序控制I/O） = 厨师每次需要用一袋面粉，都得放下手中的铲子，亲自去仓库搬一袋回来，再做菜——效率极低；
有DMA = 请一个专门的\"搬运工\"（DMA控制器），厨师只需要说：\"把100袋面粉从仓库搬到操作台\"，然后继续做菜。搬运工用最短的路径（总线主控）完成搬运，搬完再通知厨师（中断）。

图4：DMA数据传输架构 —— DMA控制器直接掌管总线，在内存和I/O设备之间建立直通数据通路，CPU只需初始化DMA参数和响应完成中断

三种I/O传输方式对比

传输方式数据单位CPU参与度适用场景速度
程序查询（轮询）字节/字全程参与（忙等）简单低速设备很慢
中断驱动字节/字每次传输参与（保存/恢复现场）中速设备（键盘、鼠标）中等
DMA数据块（批量）仅初始化+结束处理高速设备（磁盘、网卡、GPU）很快

DMA的三种传输模式

模式说明CPU状态
周期挪用DMA在CPU不访问总线时插入一个总线周期，偷一个时钟周期传输一个字几乎无影响（微暂停）
突发传输DMA连续占用总线，传输完整数据块，再释放总线完全让出总线（暂停）
透明传输DMA只在CPU不使用总线时传输（如CPU在Cache命中时）完全无影响

🔍 DMA vs 中断：核心区别

中断：每次传输一个字/字节，CPU都要保存/恢复现场，开销大但CPU能处理复杂操作。

DMA：批量传输整个数据块，每传输一个字只消耗一个总线周期，CPU完全不参与数据传输过程。传输完成后DMA发出一次中断通知CPU。

一句话：中断是\"按次服务\"，DMA是\"批量外包\"。硬盘读写大文件用DMA，键盘按键用中断。

🔧 I/O接口与编址——CPU怎么\"看见\"设备？

🏢 类比：公司和分公司的地址系统

I/O编址方式可以类比一个公司的办公地址策略：
统一编址（内存映射） = 分公司和总部在同一个大楼里，每个房间号是连续的（设备寄存器也是内存地址空间的一部分）。找分公司就找对应的房间号，MOV 指令既能访问内存也能访问设备。
独立编址 = 分公司在大楼外面不同的地址，需要专门的\"公车\"（IN/OUT 指令）才能到达。内存用一套地址，I/O设备用另一套地址，互不干扰。

图5：统一编址（Memory-Mapped）vs 独立编址（Isolated I/O）对比，以及I/O接口的四大功能模块

统一编址 vs 独立编址 详细对比

对比项统一编址（Memory-Mapped I/O）独立编址（Isolated I/O / Port-Mapped I/O）
地址空间I/O与主存共用同一地址空间I/O与主存有独立地址空间
访问指令全部内存指令（MOV, LOAD, STORE）专用 I/O 指令（IN, OUT）
地址线占用内存地址独立 I/O 地址线（或 I/O 空间选择信号）
编址容量受内存地址空间限制独立编址空间（x86: 64KB I/O端口）
优点指令丰富，C语言可直接操作（如指针）；无需专用指令不占用内存空间；地址译码简单
缺点占用内存地址范围；Cache可能缓存I/O需专用IN/OUT指令；编程不直观
典型架构ARM, RISC-V, MIPSx86 / x86-64（保留传统I/O端口）

⏱️ 程序控制I/O

  • CPU主动查询设备状态
  • 无条件: 直接执行I/O指令
  • 条件: 先查状态寄存器，设备准备就绪再传输
  • CPU利用率低（忙等）

🔄 中断驱动I/O

  • 设备就绪后主动通知CPU
  • CPU可执行其他任务
  • 每个字/字节一次中断
  • 适合中低速设备

I/O接口的主要寄存器

寄存器类型作用方向 (CPU视角)
数据寄存器 (DATA)暂存CPU和设备间交换的数据读/写
状态寄存器 (STATUS)反映设备当前状态（就绪/忙/错误）只读
控制寄存器 (CONTROL)CPU向设备发送控制命令（启动/停止/复位）只写

❓ 实战问答

Q1

总线仲裁的三种方式分别是什么？各自优缺点？

① 链式查询（菊花链）：仲裁器发出总线允许信号，设备依次传递。优点：电路简单、连线少；缺点：优先级固定，靠近仲裁器的设备总是优先，且一个设备故障会导致整条链断开。
② 计数器查询：仲裁器通过计数器轮询设备地址。优点：优先级可编程（改变计数器初值），容错性好；缺点：需要额外的地址线，速度较慢。
③ 独立请求：每个设备独立请求线和允许线，仲裁器用优先级编码器判决。优点：速度最快，优先级最灵活；缺点：连线最多，电路最复杂。现代高性能系统普遍使用独立请求方式。

Q2

中断处理的全流程是怎样的？

完整流程如下：
① 中断请求：设备向CPU发送INTR信号。
② 中断判优：若同时有多个中断，按优先级排隊（硬件实现）。
③ 中断响应：CPU发出INTA信号，自动关中断（IF=0），防止响应过程中被其他中断打断。
④ 保护断点：将当前PC（程序计数器）和PSW（程序状态字）压栈，保存现场。
⑤ 中断服务：读取中断向量表（IVT），跳转到对应的中断服务程序执行。
⑥ 恢复断点：从堆栈恢复PC和PSW，开中断，返回被中断的程序继续执行。
其中步骤④之后可选择性开中断，允许更高优先级的中断嵌套（中断嵌套）。

Q3

DMA和中断驱动I/O的核心区别是什么？什么场景用DMA？

核心区别在于数据传输的发起者和参与者：
• 中断驱动：每次传输一个字/字节，CPU都要处理一次中断（保存/恢复现场等开销）。适用于中低速设备（键盘、鼠标、串口），数据量小但要求及时响应。
• DMA：DMA控制器直接掌管总线，在内存和设备间批量传输整个数据块，CPU只需初始化DMA参数（起始地址、字数、方向）和处理传输完成中断。适用于高速大块数据场景：磁盘读写、网卡收发数据包、GPU数据传输、音频/视频流。
一句话：按次服务用中断，批量外包用DMA。键盘敲一个字用中断，硬盘读1MB文件用DMA。

Q4

统一编址（Memory-Mapped I/O）和独立编址（Isolated I/O）各有什么优缺点？

统一编址：优点——① 可以使用所有内存指令操作I/O（指令丰富），② C语言中可直接用指针访问设备寄存器，③ 不需要单独的I/O指令。缺点——① 占用内存地址空间，② 内存的Cache机制可能错误缓存I/O寄存器（需要特殊标记为uncacheable）。
独立编址：优点——① 不占用内存地址空间，② 地址译码电路简单。缺点——① 需专用IN/OUT指令，编程不灵活，② x86下I/O端口空间仅64KB。
实际选择：ARM、RISC-V等RISC架构统一用Memory-Mapped I/O（简化设计）；x86保留传统IN/OUT指令，但也大量使用Memory-Mapped I/O（如PCIe配置空间、MMIO区域）。现代架构往往是两者混用。

Q5

DMA有哪三种传输模式？各自如何工作？

① 周期挪用（Cycle Stealing）：DMA控制器在CPU不访问总线时，\"偷\"一个总线周期传输一个字。对CPU执行几乎无影响（如果Cache命中率高）。适合少量数据的定期传输。
② 突发传输（Burst Transfer）：DMA连续占用总线，传输完整个数据块后才释放。CPU在此期间完全让出总线（暂停执行）。适合大块数据传输（如磁盘读取一个扇区）。
③ 透明传输（Transparent Transfer）：DMA只在CPU的内部操作（如Cache命中、执行内部指令）不需要使用总线时传输数据。对CPU性能完全无影响，但实现复杂，且传输速率受限于CPU\"让出\"总线的时间。
实际系统中，周期挪用和突发传输最常用。现代DMA控制器通常支持多种模式混合使用。

Q6

计算机外部设备为什么需要I/O接口（接口控制器）？直接连到总线不行吗？

不行，因为CPU/系统和外设之间存在多个矛盾，I/O接口负责解决：
① 速度匹配：CPU处理速度纳秒级，机械硬盘毫秒级，打印机秒级。接口用数据缓冲寄存器/FIFO来暂存数据，让快的等慢的。
② 信号格式转换：CPU并行二进制，外设可能是串行（鼠标USB）、模拟信号（麦克风）、差分信号（SATA），接口负责电平转换、并/串转换、模/数转换。
③ 通信协议适配：CPU有自己的一套总线时序（地址/数据/控制信号时序），外设通常有自己的协议（如USB的包结构、I²C的START/STOP条件），接口作为协议翻译官。
④ 中断/DMA管理：接口负责发出中断请求、支持DMA传输信号。

朋友，我们来总结一下总线与I/O系统的核心知识。

总线与I/O系统，是计算机的\"物流和信息高速公路\"。

第一，总线分类。数据总线双向传输数据，宽度决定一次能搬多少；地址总线单向指定位置，宽度决定能编多少地址；控制总线负责指挥交通。从系统总线到I/O总线再到通信总线，形成三级总线体系。

第二，总线仲裁。多个设备要同时用总线，三种方式来解决：链式查询（菊花链）最简单但固定优先；计数器查询灵活可编程；独立请求最快但连线最多。

第三，中断系统。设备有事主动通知CPU，比CPU去\"轮询\"高效百倍。完整的流程是：请求→判优→响应→保护断点→服务→恢复断点。再用中断屏蔽字实现灵活的中断嵌套控制。

第四，DMA。让专门的控制器负责大批量数据传输，CPU只需要告诉DMA\"从哪到哪、传多少\"，然后去干别的事。硬盘读写大文件、网卡收发数据包，全都在用DMA。

第五，I/O接口与编址。统一编址把所有设备当作内存来访问，简洁统一；独立编址用专用指令访问专用地址空间，互不干扰。两种各有千秋，现代CPU往往两者兼用。
