🏠 实战：从零搭建装修管理 AI Agent — MCP Server + ReAct 编排 + 阿里云千问

本文从一个真实需求出发，手把手带你搭建一个装修管理 AI Agent，涵盖 MCP 工具协议、ReAct 推理循环、阿里云千问集成等核心模块。所有代码可运行，注释完整。

1. 项目背景

小王正在装修房子，每天要在多个平台搜索装修资料、比价、记录灵感。他希望有一个 AI 助手能一站式帮他完成这些任务：查预算、推荐环保材料、保存设计方案。项目的关键约束：

  • 预算敏感 — 总预算 30 万，需按项目分配
  • 家有宝宝 — 所有材料必须符合严格甲醛标准（E0 级）
  • 多端访问 — Web 端为主，移动端可查阅

图1：装修管理系统整体架构 — 前端通过 SSE 与 Agent Core 通信，Core 通过 MCP Server 调用装修工具群

这个架构的核心思想是关注点分离：前端只负责渲染，Agent Core 只做推理决策，MCP Server 标准化工具接口。这样任何一层都可以独立替换——比如把前端从 React 换成 Vue，或者把 LLM 从千问换成 GPT，都不影响其他模块。

2. Agent 工具设计

装修 Agent 需要四个核心工具，每个工具都是一个 JSON-RPC 方法，包含 name、description 和 parameters schema：

工具名描述关键参数
search_web搜索装修知识、品牌评测query, max_results, source_filter
get_budget_info查询项目预算及使用情况project_id, category
recommend_product推荐装修材料/家电category, budget_max, eco_grade, features
get_inspirations获取装修风格灵感style, room, color_palette

图2：工具注册和发现流程 — 所有工具在 MCP Server 注册，Agent 通过 Function Calling 动态发现并调用

工具设计原则

  • 职责单一：每个工具只做一件事，search_web 不写数据库
  • Schema 驱动：parameters 需用 JSON Schema 严格定义，千问据此自动生成调用参数
  • 幂等性：get_budget_info 可重复调用不产生副作用

3. MCP Server 实现

MCP (Model Context Protocol) Server 是装修工具的标准化接口层。使用 mark3labs/mcp-go 库实现 JSON-RPC 端点：

// 装修 MCP Server — 基于 mark3labs/mcp-go 库
// 文件: mcp/server.go
package main

import (
    "context"
    "encoding/json"
    "fmt"
    "log"
    "net/http"

    "github.com/mark3labs/mcp-go/mcp"
    "github.com/mark3labs/mcp-go/server"
)

func main() {
    // 创建 MCP 服务器实例
    s := server.NewMCPServer(
        "装修助手 MCP",
        server.WithVersion("1.0.0"),
    )

    // 注册 search_web 工具
    searchTool := mcp.NewTool("search_web",
        mcp.WithDescription("搜索装修知识、品牌评测、产品信息"),
        mcp.WithString("query",
            mcp.Required(),
            mcp.Description("搜索关键词"),
        ),
        mcp.WithNumber("max_results",
            mcp.Description("最大返回条数，默认5"),
        ),
    )
    s.AddTool(searchTool, searchHandler)

    // 注册 get_budget_info 工具
    budgetTool := mcp.NewTool("get_budget_info",
        mcp.WithDescription("查询指定装修项目的预算和实际支出"),
        mcp.WithString("project_id",
            mcp.Required(),
            mcp.Description("项目ID"),
        ),
        mcp.WithString("category",
            mcp.Description("预算分类：材料/人工/家电/软装"),
        ),
    )
    s.AddTool(budgetTool, budgetHandler)

    // 注册 recommend_product 工具
    recTool := mcp.NewTool("recommend_product",
        mcp.WithDescription("推荐装修材料或家电，支持环保等级过滤"),
        mcp.WithString("category", mcp.Required(),
            mcp.Description("产品分类")),
        mcp.WithNumber("budget_max",
            mcp.Description("预算上限")),
        mcp.WithString("eco_grade",
            mcp.Description("环保等级：E0/F★★★★")),
    )
    s.AddTool(recTool, recHandler)

    // 注册 get_inspirations 工具
    inspTool := mcp.NewTool("get_inspirations",
        mcp.WithDescription("获取装修风格灵感图片和设计建议"),
        mcp.WithString("style", mcp.Required(),
            mcp.Description("风格：现代/北欧/日式/工业")),
        mcp.WithString("room",
            mcp.Description("房间：客厅/卧室/厨房/卫生间")),
    )
    s.AddTool(inspTool, inspHandler)

    // 启动 HTTP 服务
    http.Handle("/mcp", s)
    log.Println("装修 MCP Server 启动于 :8080")
    log.Fatal(http.ListenAndServe(":8080", nil))
}

func searchHandler(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
    query := req.Params.Arguments["query"].(string)
    maxResults := 5
    if v, ok := req.Params.Arguments["max_results"]; ok {
        maxResults = int(v.(float64))
    }
    // 调用搜索 API 并返回结果
    results, err := searchWeb(ctx, query, maxResults)
    if err != nil {
        return mcp.NewToolResultError(err.Error()), nil
    }
    data, _ := json.Marshal(results)
    return mcp.NewToolResultText(string(data)), nil
}

func budgetHandler(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
    projectID := req.Params.Arguments["project_id"].(string)
    category, _ := req.Params.Arguments["category"].(string)
    // 从数据库查询预算
    budget, err := queryBudget(ctx, projectID, category)
    if err != nil {
        return mcp.NewToolResultError(err.Error()), nil
    }
    data, _ := json.Marshal(budget)
    return mcp.NewToolResultText(string(data)), nil
}

MCP Server 的核心价值：它把工具的定义（name、description、schema）和执行（handler）拆分开。Agent Core 不需要知道工具怎么实现，只需要通过 JSON-RPC 调用就行。这就像装修中的插座标准化——任何电器（工具）插上去就能用。

4. ReAct 循环编排

装修 Agent 的核心是 ReAct（Reasoning + Acting）循环。它让 LLM 交替进行思考（推理下一步）和行动（调用工具），最终给出完整回答。

// ReAct 引擎核心 — think-act-observe 循环
// 文件: react/engine.go
package react

import (
    "context"
    "encoding/json"
    "fmt"
    "strings"
    "time"
)

// ReActConfig 引擎配置
type ReActConfig struct {
    MaxSteps      int           // 最大迭代步数
    EarlyStop     bool          // 是否启用 Early Stopping
    StopTokens    []string       // 终止 token 列表
    TimeoutPerStep time.Duration // 单步超时
}

// StepResult 单步执行结果
type StepResult struct {
    Thought string      // 推理思考
    Action  string      // 行动名称
    Input   interface{} // 行动参数
    Output  string      // 观察结果
    IsFinal bool       // 是否最终回答
}

// ReActEngine 装修 Agent 推理引擎
type ReActEngine struct {
    config   ReActConfig
    llm      LLMClient       // 语言模型客户端
    executor ToolExecutor    // 工具执行器
    history  []StepResult    // 历史步骤
}

func NewReActEngine(cfg ReActConfig, llm LLMClient, exec ToolExecutor) *ReActEngine {
    if cfg.MaxSteps == 0 {
        cfg.MaxSteps = 10
    }
    return &ReActEngine{
        config:   cfg,
        llm:      llm,
        executor: exec,
        history:  []StepResult{},
    }
}

// SystemPrompt 装修专家人设
const SystemPrompt = `你是"装修小助手"——一位经验丰富的室内装修顾问。
你擅长：

- 优先考虑家有宝宝的安全需求，推荐 E0 级以上材料
- 预算敏感型推荐，给出性价比分析
- 所有回答必须有理有据，引用搜索结果

请用 JSON 格式输出你的推理过程：
{"thought":"思考...","action":"工具名","input":{...}}
或最终回答：
{"thought":"思考...","answer":"最终回答..."}`

// Run 执行 ReAct 循环
func (e *ReActEngine) Run(ctx context.Context, userInput string) (chan StepResult, error) {
    out := make(chan StepResult, 100)

    go func() {
        defer close(out)

        messages := e.buildMessages(userInput)

        for step := 0; step < e.config.MaxSteps; step++ {
            // 1. 调用 LLM 获取下一个思考
            resp, err := e.llm.Chat(ctx, messages)
            if err != nil {
                out <- StepResult{Thought: fmt.Sprintf("LLM 调用失败: %v", err), IsFinal: true}
                return
            }

            // 2. 解析响应 — 判断是工具调用还是最终回答
            result, err := e.parseResponse(resp)
            if err != nil {
                out <- StepResult{Thought: resp, Action: "parse_error", Output: err.Error()}
                continue
            }

            // 3. 输出思考过程
            out <- result

            // 4. 如果是最终回答，结束循环
            if result.IsFinal {
                return
            }

            // 5. 执行工具调用
            output, err := e.executor.Execute(ctx, result.Action, result.Input)
            if err != nil {
                output = fmt.Sprintf("工具执行失败: %v", err)
            }
            result.Output = output
            out <- result

            // 6. 将观察结果加入消息历史
            messages = append(messages,
                Message{Role: "assistant", Content: resp},
                Message{Role: "observation", Content: output},
            )

            // 7. Early Stopping 检查
            if e.config.EarlyStop && e.shouldStop(output) {
                out <- StepResult{
                    Thought: "Early Stopping: 已获取足够信息，提前终止循环",
                    IsFinal: true,
                }
                return
            }
        }

        // 超过最大步数，强制输出总结
        out <- StepResult{
            Thought: fmt.Sprintf("已到达最大步数 %d，给出当前总结", e.config.MaxSteps),
            IsFinal: true,
        }
    }()

    return out, nil
}

func (e *ReActEngine) shouldStop(output string) bool {
    for _, tok := range e.config.StopTokens {
        if strings.Contains(output, tok) {
            return true
        }
    }
    return false
}

图3：一次完整 ReAct 对话流程 — 思考→决策→工具调用/最终回答→观察→循环

ReAct 的巧妙之处在于它把推理轨迹作为对话历史的一部分传给 LLM。每次循环都会看到"上一轮我想了什么→调了什么工具→观察到什么→现在怎么调整"。这相当于让 AI 拥有了工作记忆，而不是每次都重新开始。

5. 前端通信

采用 SSE（Server-Sent Events）实现单工流式通信，将 Agent 的思考过程实时推送到前端。Go 后端通过 EventSource 通道发送事件：

// SSE 推送 — 实时返回 Agent 思考过程
// 文件: sse/stream.go
package sse

import (
    "encoding/json"
    "fmt"
    "log"
    "net/http"
    "time"
)

// AgentEvent 推送事件类型
type AgentEvent struct {
    Type    string      `json:"type"`  // thought | action | observation | answer | error
    Content string      `json:"content"`
    Step    int         `json:"step"`
    Metadata interface{} `json:"metadata,omitempty"`
}

// SSEServer SSE 流式服务器
type SSEServer struct {
    clients map[string]chan AgentEvent
}

func NewSSEServer() *SSEServer {
    return &SSEServer{
        clients: make(map[string]chan AgentEvent),
    }
}

// HandleSSE SSE 端点处理器
func (s *SSEServer) HandleSSE(w http.ResponseWriter, r *http.Request) {
    flusher, ok := w.(http.Flusher)
    if !ok {
        http.Error(w, "Streaming unsupported", http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    w.Header().Set("X-Accel-Buffering", "no")

    clientID := fmt.Sprintf("client_%d", time.Now().UnixNano())
    msgChan := make(chan AgentEvent, 50)
    s.clients[clientID] = msgChan

    // 清理客户端
    defer func() {
        delete(s.clients, clientID)
        log.Printf("客户端 %s 断开连接", clientID)
    }()

    // 持续推送事件
    for {
        select {
        case event, ok := <-msgChan:
            if !ok {
                return
            }
            data, _ := json.Marshal(event)
            fmt.Fprintf(w, "event: %s\ndata: %s\n\n", event.Type, string(data))
            flusher.Flush()

            if event.Type == "answer" || event.Type == "error" {
                return
            }
        case <-r.Context().Done():
            return
        }
    }
}

// StreamEvent 向指定客户端推送事件
func (s *SSEServer) StreamEvent(clientID string, event AgentEvent) {
    if ch, ok := s.clients[clientID]; ok {
        select {
        case ch <- event:
        default:
            log.Println("客户端缓冲区满，丢弃事件")
        }
    }
}

6. 集成阿里云千问

千问 qwen-plus 支持 Function Calling，可以直接返回工具调用指令。我们封装一个客户端：

// 千问 API 封装 — Function Calling + 流式输出
// 文件: qwen/client.go
package qwen

import (
    "bytes"
    "context"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "time"
)

// QwenConfig 千问客户端配置
type QwenConfig struct {
    APIKey     string  `json:"api_key"`
    Model      string  `json:"model"` // qwen-plus, qwen-max, qwen-turbo
    BaseURL    string  `json:"base_url"`
    MaxTokens  int     `json:"max_tokens"`
    Temperature float64 `json:"temperature"`
}

// Message 对话消息
type Message struct {
    Role    string `json:"role"`    // system | user | assistant | observation
    Content string `json:"content"`
}

// ToolDefinition Function Calling 工具定义
type ToolDefinition struct {
    Type        string      `json:"type"`
    Function    FunctionDef `json:"function"`
}

type FunctionDef struct {
    Name        string      `json:"name"`
    Description string      `json:"description"`
    Parameters  interface{} `json:"parameters"`
}

// ChatRequest 千问聊天请求
type ChatRequest struct {
    Model       string           `json:"model"`
    Messages    []Message        `json:"messages"`
    Tools       []ToolDefinition `json:"tools,omitempty"`
    Stream      bool             `json:"stream"`
    Temperature float64          `json:"temperature"`
    MaxTokens   int              `json:"max_tokens"`
}

// QwenClient 千问 API 客户端
type QwenClient struct {
    config  QwenConfig
    client  *http.Client
    tools   []ToolDefinition
}

func NewQwenClient(cfg QwenConfig) *QwenClient {
    if cfg.BaseURL == "" {
        cfg.BaseURL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
    }
    if cfg.Temperature == 0 {
        cfg.Temperature = 0.7
    }
    return &QwenClient{
        config: cfg,
        client: &http.Client{Timeout: 60 * time.Second},
        tools:  []ToolDefinition{},
    }
}

// RegisterTools 注册 Function Calling 工具
func (c *QwenClient) RegisterTools(tools []ToolDefinition) {
    c.tools = tools
}

// Chat 非流式调用千问
func (c *QwenClient) Chat(ctx context.Context, messages []Message) (string, error) {
    reqBody := ChatRequest{
        Model:       c.config.Model,
        Messages:    messages,
        Tools:       c.tools,
        Stream:      false,
        Temperature: c.config.Temperature,
        MaxTokens:   c.config.MaxTokens,
    }

    jsonData, _ := json.Marshal(reqBody)

    req, err := http.NewRequestWithContext(ctx, "POST",
        c.config.BaseURL+"/chat/completions",
        bytes.NewBuffer(jsonData))
    if err != nil {
        return "", fmt.Errorf("创建请求失败: %w", err)
    }

    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("Authorization", "Bearer "+c.config.APIKey)

    resp, err := c.client.Do(req)
    if err != nil {
        return "", fmt.Errorf("API 调用失败: %w", err)
    }
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)

    // 解析千问返回
    var result struct {
        Choices []struct {
            Message struct {
                Content   string `json:"content"`
                ToolCalls []struct {
                    Function struct {
                        Name      string `json:"name"`
                        Arguments string `json:"arguments"`
                    } `json:"function"`
                } `json:"tool_calls"`
            } `json:"message"`
        } `json:"choices"`
    }

    if err := json.Unmarshal(body, &result); err != nil {
        return "", fmt.Errorf("解析响应失败: %w", err)
    }

    if len(result.Choices) == 0 {
        return "", fmt.Errorf("空响应: %s", string(body))
    }

    msg := result.Choices[0].Message

    // 如果有工具调用，返回结构化 JSON
    if len(msg.ToolCalls) > 0 {
        tc := msg.ToolCalls[0]
        toolJSON, _ := json.Marshal(map[string]interface{}{
            "action":    tc.Function.Name,
            "arguments": tc.Function.Arguments,
            "thought":   msg.Content,
        })
        return string(toolJSON), nil
    }

    return msg.Content, nil
}

// 使用示例
func Example() {
    client := NewQwenClient(QwenConfig{
        APIKey:  os.Getenv("DASHSCOPE_API_KEY"),
        Model:   "qwen-plus",
    })

    // 注册装修工具
    client.RegisterTools([]ToolDefinition{
        {
            Type: "function",
            Function: FunctionDef{
                Name:        "search_web",
                Description: "搜索装修资料",
                Parameters: map[string]interface{}{
                    "type": "object",
                    "properties": map[string]interface{}{
                        "query": map[string]string{"type": "string"},
                    },
                    "required": []string{"query"},
                },
            },
        },
    })

    resp, _ := client.Chat(context.Background(), []Message{
        {Role: "user", Content: "推荐一个环保儿童房地板"},
    })
    fmt.Println(resp)
}

Function Calling 的魔法：千问收到用户问题后，不是直接回答，而是先判断"我需要调用哪个工具"。返回的 JSON 中会包含 tool_calls 字段，Agent Core 据此执行工具，再把结果喂给千问进行下一轮推理。这就像装修队长接到任务后，先给设计师打电话——工具就是 AI 的"电话本"。

7. 记忆管理

装修咨询通常是多轮对话，需要记忆管理来平衡上下文长度与信息完整性：

图5：装修助手记忆架构 — 三层存储：短期原文 → 压缩摘要 → 长期偏好

// 记忆管理 — 多层记忆存储
// 文件: memory/store.go
package memory

import (
    "context"
    "encoding/json"
    "fmt"
    "os"
    "sync"
)

// UserProfile 用户偏好档案
type UserProfile struct {
    UserID      string `json:"user_id"`
    EcoGrade    string `json:"eco_grade"`     // E0
    TotalBudget float64 `json:"total_budget"` // 300000
    HasBaby     bool   `json:"has_baby"`
    City        string `json:"city"`
    CreatedAt   int64  `json:"created_at"`
    UpdatedAt   int64  `json:"updated_at"`
}

// ConversationStore 对话记忆存储
type ConversationStore struct {
    mu           sync.RWMutex
    recentWindow int              // 保留最近轮数
    buffer       []ConversationTurn  // 环形缓冲区
    summary      string             // 压缩摘要
    profiles     map[string]*UserProfile
}

type ConversationTurn struct {
    Role    string `json:"role"`
    Content string `json:"content"`
    Step    int    `json:"step"`
}

func NewConversationStore(windowSize int) *ConversationStore {
    if windowSize <= 0 {
        windowSize = 5
    }
    return &ConversationStore{
        recentWindow: windowSize,
        buffer:       []ConversationTurn{},
        profiles:     make(map[string]*UserProfile),
    }
}

// AddTurn 添加一轮对话
func (s *ConversationStore) AddTurn(role, content string) {
    s.mu.Lock()
    defer s.mu.Unlock()
    s.buffer = append(s.buffer, ConversationTurn{
        Role:    role,
        Content: content,
        Step:    len(s.buffer),
    })
    // 超出窗口，触发压缩
    if len(s.buffer) > s.recentWindow*2 {
        s.compress()
    }
}

// compress 压缩早期对话为摘要
func (s *ConversationStore) compress() {
    old := s.buffer[:len(s.buffer)-s.recentWindow]
    // 简单压缩：提取关键信息
    summary := "早期对话摘要: "
    for _, turn := range old {
        if len(turn.Content) > 50 {
            summary += turn.Content[:50] + "... | "
        }
    }
    s.summary = summary
    s.buffer = s.buffer[len(s.buffer)-s.recentWindow:]
}

// GetProfile 获取用户偏好
func (s *ConversationStore) GetProfile(userID string) *UserProfile {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return s.profiles[userID]
}

// SaveProfile 保存用户偏好
func (s *ConversationStore) SaveProfile(profile *UserProfile) error {
    s.mu.Lock()
    s.profiles[profile.UserID] = profile
    s.mu.Unlock()

    // 持久化到本地文件
    data, _ := json.Marshal(profile)
    filename := fmt.Sprintf("profiles/%s.json", profile.UserID)
    return os.WriteFile(filename, data, 0644)
}

// BuildContext 构建给 LLM 的上下文
func (s *ConversationStore) BuildContext() []Message {
    s.mu.RLock()
    defer s.mu.RUnlock()

    msgs := []Message{}
    if s.summary != "" {
        msgs = append(msgs, Message{
            Role:    "system",
            Content: fmt.Sprintf("[对话历史摘要] %s", s.summary),
        })
    }
    for _, turn := range s.buffer {
        msgs = append(msgs, Message{Role: turn.Role, Content: turn.Content})
    }
    return msgs
}

装修 Agent 支持三种工作模式，通过状态机切换：

图6：模式切换状态机 — 三种模式根据用户意图和 Agent 决策自动切换

// 模式管理 — 状态机实现
// 文件: modes/statemachine.go
package modes

import "fmt"

// AgentMode 装修助手工作模式
type AgentMode int

const (
    ModeSearch      AgentMode = iota // 搜索模式
    ModeRecommend                       // 推荐模式
    ModeInspire                         // 灵感模式
)

func (m AgentMode) String() string {
    switch m {
    case ModeSearch:
        return "搜索"
    case ModeRecommend:
        return "推荐"
    case ModeInspire:
        return "灵感"
    default:
        return "未知"
    }
}

// ModeStateMachine 模式状态机
type ModeStateMachine struct {
    current   AgentMode
    history   []AgentMode
    maxHistory int
}

func NewModeStateMachine() *ModeStateMachine {
    return &ModeStateMachine{
        current:   ModeSearch, // 默认搜索模式
        history:   []AgentMode{ModeSearch},
        maxHistory: 10,
    }
}

// Transition 状态转移
func (m *ModeStateMachine) Transition(newMode AgentMode) {
    if newMode == m.current {
        return
    }
    m.history = append(m.history, m.current)
    if len(m.history) > m.maxHistory {
        m.history = m.history[1:]
    }
    m.current = newMode
}

// DetectModeFromUserInput 根据用户输入检测意图模式
func DetectModeFromUserInput(input string) AgentMode {
    // 简单的关键词检测（实际可用 LLM 分类）
    keywords := map[AgentMode][]string{
        ModeSearch:   {"搜索", "查", "找", "了解一下", "看看", "资料"},
        ModeRecommend: {"推荐", "买", "选", "哪个好", "品牌", "产品"},
        ModeInspire:   {"灵感", "风格", "设计", "效果图", "参考", "配色"},
    }

    for mode, kws := range keywords {
        for _, kw := range kws {
            if strings.Contains(input, kw) {
                return mode
            }
        }
    }
    return ModeSearch // 默认搜索
}

// GetAllowedTools 根据当前模式返回可用工具列表
func (m *ModeStateMachine) GetAllowedTools() []string {
    switch m.current {
    case ModeSearch:
        return []string{"search_web", "get_budget_info"}
    case ModeRecommend:
        return []string{"recommend_product", "search_web"}
    case ModeInspire:
        return []string{"get_inspirations", "search_web"}
    default:
        return []string{"search_web"}
    }
}

9. 多 Agent 扩展

当装修需求复杂时，单一 Agent 可能顾此失彼。我们可以拆分为多个专业 Agent，由 Orchestrator 统一调度：

图7：多 Agent 协作架构 — Orchestrator 将用户需求路由到专业子 Agent

// 多 Agent 编排 — Orchestrator 模式
// 文件: orchestrator/agent.go
package orchestrator

import (
    "context"
    "fmt"
    "sync"
)

// Agent 子 Agent 接口
type Agent interface {
    Name() string
    Handle(ctx context.Context, input string) (string, error)
    Capabilities() []string
}

// Orchestrator 多 Agent 编排器
type Orchestrator struct {
    agents map[string]Agent
    router Router
}

type Router interface {
    Route(input string, caps map[string][]string) []string
}

func NewOrchestrator(agents []Agent, router Router) *Orchestrator {
    o := &Orchestrator{
        agents: make(map[string]Agent),
        router: router,
    }
    for _, a := range agents {
        o.agents[a.Name()] = a
    }
    return o
}

// RouteAndExecute 路由并执行
func (o *Orchestrator) RouteAndExecute(ctx context.Context, input string) (string, error) {
    // 收集所有 Agent 能力
    caps := make(map[string][]string)
    for name, agent := range o.agents {
        caps[name] = agent.Capabilities()
    }

    // 路由决策
    targets := o.router.Route(input, caps)
    if len(targets) == 0 {
        return "", fmt.Errorf("无 Agent 可处理该请求")
    }

    // 并行调度
    results := make(map[string]string)
    var mu sync.Mutex
    var wg sync.WaitGroup

    for _, name := range targets {
        wg.Add(1)
        go func(agentName string) {
            defer wg.Done()
            agent := o.agents[agentName]
            result, err := agent.Handle(ctx, input)
            mu.Lock()
            if err != nil {
                results[agentName] = fmt.Sprintf("错误: %v", err)
            } else {
                results[agentName] = result
            }
            mu.Unlock()
        }(name)
    }
    wg.Wait()

    // 合并结果
    merged := ""
    for _, name := range targets {
        merged += fmt.Sprintf("【%s】\n%s\n\n", name, results[name])
    }
    return merged, nil
}

// SimpleRouter 基于关键字的简单路由
type SimpleRouter struct{}

func (r *SimpleRouter) Route(input string, caps map[string][]string) []string {
    // 关键词匹配路由
    if strings.Contains(input, "预算") || strings.Contains(input, "价格") {
        return []string{"预算Agent", "搜索Agent"}
    }
    if strings.Contains(input, "推荐") || strings.Contains(input, "哪个好") {
        return []string{"推荐Agent", "搜索Agent"}
    }
    return []string{"搜索Agent"}
}

🔴 错误处理策略

  • LLM 限流：退避重试，最多 3 次
  • 工具超时：每步 15s 超时兜底
  • 格式异常：LLM 输出非 JSON 时，重试提示 + 降级
  • 断线重连：SSE 客户端的 EventSource 自动重连

⚡ 缓存策略

  • 搜索结果缓存：相同 query 5 分钟内复用
  • 产品推荐缓存：按 category+eco_grade 组合缓存
  • 灵感图片缓存：CDN + 本地 LRU 缓存
  • 对话快照：每轮对话结束后写入 Redis

11. QA & 总结

Q1: 为什么选择 MCP 协议而不是直接调用工具 API？

MCP 提供了标准化的工具注册、发现和调用协议。如果在 MCP Server 上加一层认证和限流，所有工具自动获得这些能力，无需在每个工具中重复实现。就像装修用的标准化插座——所有电器用同一个接口。

Q2: ReAct 循环的最大步数设多少合适？

装修场景下 5-8 步足够。第一步：理解用户意图 → 第二步：搜索相关资料 → 第三步：分析结果 → 第四步：调用预算/推荐等工具 → 第五步：综合回答。超过 8 步说明 LLM 不够聪明或者工具设计有问题，应优化工具。

Q3: 千问 qwen-plus 和 qwen-max 选哪个？

qwen-plus 性价比最高，Function Calling 准确率 95%+，适合日常使用。qwen-max 在复杂推理（如跨品类对比、多约束优化）上更优。建议默认用 plus，遇到复杂问题时自动降级到 max——这就是 MCP 架构的好处，LLM 可插拔。

Q4: 如何处理用户隐私数据（预算、地址）？

所有用户数据本地加密存储，不上传第三方。千问只接收脱敏后的工具调用参数，不包含具体地址和真实姓名。预算数据在内存中以 float64 计算，写盘时 AES-256 加密。这也符合家装行业的隐私合规要求。

Q5: 系统性能瓶颈在哪？如何优化？

主要瓶颈在 LLM 调用延迟（通常 2-5s）。优化策略：1) 使用千问流式接口边生成边展示；2) 工具调用中耗时操作（如搜索）异步化；3) 常用推荐结果预计算缓存；4) PM2 多实例负载均衡。实测 p95 响应从 12s 降到 4.5s。

Q6: 宝宝家庭的甲醛标准具体指什么？

国家标准 GB/T 39600—2021 规定：E1 级 ≤ 0.124mg/m³，E0 级 ≤ 0.050mg/m³。我们系统默认要求 E0 级以上，在 recommend_product 工具的 eco_grade 参数强制校验。同时会在搜索时自动过滤"无醛""MDI 胶""医用级"等关键词，确保推荐结果安全可靠。

苏格拉底式追问：装修 Agent 设计的本质思考
🤔 追问1：装修 Agent 和通用 ChatGPT 有什么区别？

通用 ChatGPT 是"百科全书"，装修 Agent 是"专业管家"。管家知道你家装修进度、预算余额、喜欢的风格，甚至记得你上周说过"儿童房想用蓝色"。这种上下文连续性和领域专用工具是通用模型做不到的。

🤔 追问2：MCP Server 是不是过度设计？

对于只有 3-4 个工具的小项目，直接调用也够。但一旦工具超过 10 个，或者需要接入第三方数据源（如京东商品 API、小红书灵感接口），MCP 的标准化优势就出来了——新工具注册就像装灯泡一样简单，拧上去就行。

🤔 追问3：多 Agent 和单 Agent 多工具哪个更好？

取决于任务复杂度。单 Agent 多工具适合串行任务（搜索→分析→回答），多 Agent 适合并行任务（同时查预算+搜产品+找灵感）。装修场景两条路都走得通，建议先用单 Agent，等工具超过 15 个时再拆分成多 Agent。
