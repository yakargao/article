1. MCP 是什么

MCP（Model Context Protocol） 是 Anthropic 于 2024 年 11 月开源的一个开放协议，旨在标准化 AI 模型与外部工具、数据源之间的交互方式。它定义了统一的「工具发现 → 调用 → 结果返回」生命周期，让 LLM 应用可以像「即插即用」一样接入各种能力。

想象你有一台笔记本电脑，想连接打印机、显示器、键盘、鼠标……如果没有 USB 标准，你需要为每种外设准备不同的接口和驱动。MCP 就是 AI 世界的 USB-C —— 它为 AI Agent 提供了一个统一的接口标准，从此任何兼容 MCP 的工具都可以「即插即用」。

诞生背景

🔒 OpenAI Function Calling

专有协议，仅限 OpenAI 模型使用。每次新增工具需要写大量适配代码。生态封闭，存在厂商锁定风险。

2. MCP 架构概览

MCP 采用 Client-Server 架构，分为四个核心层次：

核心角色说明

🖥️ Host (宿主)

运行 LLM 的应用，比如 Claude Desktop、IDE 插件、或你的自定义 Agent 框架。Host 创建 Client 实例。

📡 Client (客户端)

与 MCP Server 建立一对一连接。负责发送 JSON-RPC 请求、接收响应、管理传输层握手。

🔌 Server (服务端)

暴露工具、数据资源和提示模板的程序。通过 Capabilities 声明自己支持的功能点。

🔧 Tool/Resource

Tool = 可执行函数（写数据库、发邮件）；Resource = 可读取数据（文件、API）；Prompt = 预定义提示模板。

4. 工具发现机制

MCP 的工具交互基于标准的 JSON-RPC 2.0 协议，分为两个核心流程：

📋 工具发现 (listTools)

Client 向 Server 发送 tools/list 请求，Server 返回可用工具列表（名称、描述、输入 Schema）。LLM 根据 Schema 决定调用哪个工具并填充参数。

📞 工具调用 (tools/call)

Client 发送 tools/call 请求，包含工具名称和参数。Server 执行并返回结果（成功数据或错误信息）。

JSON-RPC 消息格式

// listTools 请求
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}

// tools/list 响应
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [{
      "name": "search_product",
      "description": "搜索商品库存信息",
      "inputSchema": {
        "type": "object",
        "properties": {
          "query": { "type": "string" }
        },
        "required": ["query"]
      }
    }]
  }
}

// tools/call 请求
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "search_product",
    "arguments": { "query": "laptop" }
  }
}

5. Resource 与 Prompt

除了 Tool，MCP 还定义了两种能力类型，让模型能够访问数据和使用提示模板：

能力用途类比MCP 方法
Tool 执行操作（写入、计算、调用外部 API） 🔧 扳手 — 用来「做事」 tools/list tools/call
Resource 读取数据（文件、数据库、API 响应） 📂 文件柜 — 用来「看数据」 resources/list resources/read
Prompt 获取预定义提示模板 📝 模板卡片 — 用来「说」 prompts/list prompts/get

Go 代码：注册 Tool 和 Resource

package main

import (
    "context"
    "encoding/json"
    "fmt"
    "log"

    "github.com/mark3labs/mcp-go/mcp"
    "github.com/mark3labs/mcp-go/server"
)

func main() {
    // 创建 MCP 服务端
    s := server.NewMCP(server.WithName("product-server"))

    // ── 注册 Tool ──
    searchProductTool := mcp.NewTool("search_product",
        mcp.WithDescription("按关键词搜索商品库存"),
        mcp.WithString("query",
            mcp.Required(),
            mcp.Description("搜索关键词（如 'laptop'）"),
        ),
    )

    s.AddTool(searchProductTool, func(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
        query := req.Params.Arguments["query"].(string)
        result, _ := json.Marshal(searchDB(query))
        return mcp.NewToolResultText(string(result)), nil
    })

    // ── 注册 Resource ──
    s.AddResource(
        mcp.NewResource("inventory://products",
            "product-list",
            mcp.WithResourceDescription("当前商品库存列表 (CSV)"),
            mcp.WithMIMEType("text/csv"),
        ),
        func(ctx context.Context, req mcp.ReadResourceRequest) ([]mcp.ResourceContents, error) {
            csv := `id,name,stock,price
1,MacBook Pro,23,1999
2,ThinkPad X1,15,1799`
            return []mcp.ResourceContents{
                mcp.NewTextResourceContents("inventory://products",
                    "text/csv", csv),
            }, nil
        },
    )

    log.Println("MCP Server started on stdio...")
    if err := server.ServeStdio(s); err != nil {
        log.Fatal(err)
    }
}

func searchDB(q string) []map[string]any {
    // 模拟数据库查询
    return []map[string]any{
        {"id": 1, "name": "MacBook Pro M4", "stock": 23},
        {"id": 2, "name": "ThinkPad X1 Carbon", "stock": 15},
    }
}

6. MCP vs Function Calling vs API

这三者虽然都涉及「模型调用外部能力」，但抽象层级完全不同：

MCP 不是要替代 Function Calling 或 REST API，而是站在更高的抽象层次上。如果把 Function Calling 比作「插座上的国标插孔」，MCP 就是「带转换器的万能插座 — 美标、欧标、英标都能插」。传统 REST API 则是「直接拿电线接进墙里」—— 强大但需要专业电工。

7. A2A 协议

A2A（Agent-to-Agent） 是 Google 在 2025 年 4 月提出的 Agent 间通信协议，与 MCP 形成互补关系：

🔌 MCP (Model → Context)

模型与工具/数据之间的标准化接口。「垂直」集成 —— Agent 使用工具。

Tool Integration

🤝 A2A (Agent → Agent)

智能体之间的协作通信协议。「水平」集成 —— Agent 调用 Agent。

Agent Orchestration

协议栈

MCP + A2A 协同工作流

// Agent A 通过 MCP 查询数据，通过 A2A 委派任务给 Agent B
func handleUserQuery(userQuery string) {
    // 1. MCP: 从数据库读用户信息
    userData := mcpClient.readResource("db://users/123")

    // 2. LLM 推理：需要 Agent B 协助
    if needsSpecialist(userQuery) {
        // 3. A2A: 委派任务给 Agent B
        taskID := a2aClient.sendTask("agent-b@internal", Task{
            Query:   userQuery,
            Context: userData,
        })

        // 4. 轮询或等待推送获取结果
        result := a2aClient.awaitTask(taskID)
        replyToUser(result)
    }
}

package main

import (
    "context"
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "encoding/json"
    "errors"
    "fmt"
    "log"
    "net/http"
    "os"
    "time"

    "github.com/mark3labs/mcp-go/mcp"
    "github.com/mark3labs/mcp-go/server"
)

// ── 数据模型 ──
type Product struct {
    ID    int     `json:"id"`
    Name  string  `json:"name"`
    Stock int     `json:"stock"`
    Price float64 `json:"price"`
}

type Budget struct {
    Total   float64 `json:"total"`
    Used    float64 `json:"used"`
    Remains float64 `json:"remains"`
}

// ── 内存数据库 ──
var products = []Product{
    {1, "MacBook Pro M4", 23, 1999.00},
    {2, "ThinkPad X1 Carbon", 15, 1799.00},
    {3, "Dell XPS 16", 8, 1899.00},
}

var budget = Budget{Total: 100000, Used: 42350}

// ── API Key 验证中间件 ──
func authMiddleware(expectedKey string) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            key := r.Header.Get("X-API-Key")
            if key == "" || !validateHMAC(key, expectedKey) {
                http.Error(w, "unauthorized", http.StatusUnauthorized)
                return
            }
            next.ServeHTTP(w, r)
        })
    }
}

func validateHMAC(provided, secret string) bool {
    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write([]byte(provided))
    expected := hex.EncodeToString(mac.Sum(nil))
    return hmac.Equal([]byte(provided), []byte(expected))
}

// ── 工具1: search_product ──
func handleSearchProduct(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
    query, _ := req.Params.Arguments["query"].(string)
    if query == "" {
        return nil, errors.New("query is required")
    }

    var results []Product
    for _, p := range products {
        if containsIgnoreCase(p.Name, query) {
            results = append(results, p)
        }
    }

    data, _ := json.Marshal(results)
    return mcp.NewToolResultText(string(data)), nil
}

// ── 工具2: get_budget ──
func handleGetBudget(ctx context.Context, req mcp.CallToolRequest) (*mcp.CallToolResult, error) {
    budget.Remains = budget.Total - budget.Used
    data, _ := json.Marshal(budget)
    return mcp.NewToolResultText(string(data)), nil
}

func containsIgnoreCase(s, substr string) bool {
    return len(s) >= len(substr) &&
        contains(toLower(s), toLower(substr))
}

func toLower(s string) string {
    b := []byte(s)
    for i, c := range b {
        if c >= 'A' && c <= 'Z' {
            b[i] = c + 32
        }
    }
    return string(b)
}

func contains(s, substr string) bool {
    for i := 0; i <= len(s)-len(substr); i++ {
        if s[i:i+len(substr)] == substr {
            return true
        }
    }
    return false
}

// ── Main ──
func main() {
    apiKey := os.Getenv("MCP_API_KEY")
    if apiKey == "" {
        log.Fatal("MCP_API_KEY environment variable required")
    }

    // 创建 SSE 传输 MCP Server
    s := server.NewMCP(
        server.WithName("product-mcp-server"),
        server.WithVersion("1.0.0"),
    )

    // 注册 search_product 工具
    searchTool := mcp.NewTool("search_product",
        mcp.WithDescription("按关键词搜索商品库存信息"),
        mcp.WithString("query",
            mcp.Required(),
            mcp.Description("商品名称关键词"),
        ),
        mcp.WithNumber("max_results",
            mcp.Description("最多返回结果数 (默认 10)"),
        ),
    )
    s.AddTool(searchTool, handleSearchProduct)

    // 注册 get_budget 工具
    budgetTool := mcp.NewTool("get_budget",
        mcp.WithDescription("查看当前采购预算总额、已用和剩余金额"),
    )
    s.AddTool(budgetTool, handleGetBudget)

    // 启动 SSE + HTTP 服务
    mcpHandler := server.NewSSEServer(s)
    handler := authMiddleware(apiKey)(mcpHandler)

    httpServer := &http.Server{
        Addr:         ":8080",
        Handler:      handler,
        ReadTimeout:  10 * time.Second,
        WriteTimeout: 30 * time.Second,
    }

    log.Printf("MCP Server running on :8080 with API Key auth\n")
    log.Fatal(httpServer.ListenAndServe())
}

9. 常见问题 (QA)

MCP 只能在 Anthropic 的模型上使用吗？
不是。MCP 是一个开放协议，任何 LLM 都可以通过实现 MCP Client 来使用。目前已有多家框架支持：LangChain、LlamaIndex、OpenAI Agents SDK 等都有 MCP 集成。Anthropic 是发起者，但不专享。
MCP 是否支持非文本工具（图片生成、音频处理）？
可以。MCP 的 CallToolResult 支持多种 content 类型：text、image（Base64 编码）、resource（引用外部资源）、embedded（嵌入资源）。你可以返回图片、音频、二进制数据等。
MCP 和 Function Calling 到底有什么实质区别？
Function Calling 是「模型能力」—— 模型在生成响应时能调用预定义函数。MCP 是「生态协议」—— 它标准化了工具发现、注册、调用、传输全流程。你可以把 MCP 理解成「Function Calling 标准化框架」：FC 只管「调」，MCP 管「怎么找到、怎么连接、怎么调、怎么返回」。
MCP Server 一定是远程 HTTP 服务吗？
不一定。MCP 支持 stdio 传输，意味着 Server 可以作为一个本地子进程运行，通过标准输入/输出通信。这在本地开发、调试场景下非常方便，零依赖、极低延迟。Claude Desktop 就是通过 stdio 启动本地 MCP Server 的。
MCP 如何处理长时间运行的任务？
MCP 协议设计了 notifications 机制来支持异步和长时间任务。Server 可以通过 notifications/tasks/progress 通知客户端进度。对于极端长任务，标准做法是先返回一个「任务接受」响应，然后通过另一个通道（如 SSE 推送）在任务完成时通知客户端。这类似于 A2A 中使用的轮询+推送模式。
多个工具依赖（如先查库存再下单）MCP 能处理吗？
MCP 协议本身专注于单次工具调用，调用编排由 LLM 层负责。通常情况下，LLM 会先调用 search_product 获取商品 ID，再用结果作为输入调用 create_order。这个「思考 → 调用 → 思考 → 再调用」的多步流程由 LLM 的推理能力和 MCP Client 的会话管理共同完成。

10. 口播总结

🎙️ 一句话总结：

MCP（Model Context Protocol） 是 Anthropic 开源的 AI Agent 工具集成标准协议，它像 USB-C 一样统一了 LLM 与外部世界的连接方式，让 N×M 的集成爆炸简化为 N+M。

🔑 核心要点：

  • 四层架构：Host → Client → Transport → Server/Tool，每层各司其职
  • 三种传输：stdio（本地）、SSE（远程单向）、WebSocket（实时双向）
  • 三大能力：Tool（执行操作）、Resource（读取数据）、Prompt（提示模板）
  • 标准协议：基于 JSON-RPC 2.0，tools/list 发现，tools/call 调用
  • 开放生态：与 A2A（Agent 间协作）互补，一起构建完整的 Agent 互联生态
  • 快速上手：Go 社区有 mark3labs/mcp-go 库，几十行代码就能跑起一个 MCP Server

💡 展望：2025 年是 AI Agent 从「单体智能」走向「群体协作」的关键一年。MCP 统一了「模型→工具」的接口，A2A 统一了「Agent→Agent」的接口。两者的结合，正在为 AI 生态搭建一个类似互联网 TCP/IP 的基座协议栈。作为开发者，现在上手 MCP 就是抓住下一波红利的入场券。
